# TypeScript

TypeScript is a strongly typed programming language that builds on JavaScript, giving you better tooling at any scale. Developed and maintained by Microsoft, TypeScript adds optional static typing, classes, and interfaces to JavaScript, making it easier to build and maintain large-scale applications.

## Key Features

- **Static Type Checking**: Catch errors at compile time
- **Type Inference**: Smart automatic type detection
- **Modern JavaScript**: Supports latest ECMAScript features
- **Tooling**: Enhanced IDE support and autocompletion
- **Gradual Adoption**: Add types incrementally to existing JavaScript
- **Backwards Compatible**: Compiles to clean, readable JavaScript

## Type System

### Basic Types
```typescript
// Primitives
let name: string = "Alice";
let age: number = 30;
let isDeveloper: boolean = true;

// Arrays
let languages: string[] = ["TypeScript", "Python", "Go"];
let numbers: Array<number> = [1, 2, 3];

// Tuples
let person: [string, number] = ["Alice", 30];

// Enums
enum TechnologyCategory {
  ProgrammingLanguage = "Programming Language",
  Framework = "Framework",
  Database = "Database"
}

// Any (escape hatch)
let anything: any = "can be anything";

// Unknown (safer than any)
let something: unknown = "must be type-checked before use";
```

### Advanced Types
```typescript
// Interfaces
interface Technology {
  name: string;
  category: TechnologyCategory;
  hotTechnology?: boolean; // Optional
}

// Type Aliases
type UserId = string | number;

// Union Types
type Status = "active" | "inactive" | "pending";

// Intersection Types
type Developer = Person & Employee;

// Generics
function identity<T>(arg: T): T {
  return arg;
}

// Utility Types
type Partial<T> = { [P in keyof T]?: T[P] };
type Required<T> = { [P in keyof T]-?: T[P] };
type Readonly<T> = { readonly [P in keyof T]: T[P] };
```

## Example Code

```typescript
// Interface definition
interface Developer {
  name: string;
  occupation: string;
  languages: string[];
  yearsExperience: number;
}

// Function with typed parameters and return type
function introduceDeveloper(dev: Developer): string {
  return `${dev.name} is a ${dev.occupation} with ${dev.yearsExperience} years of experience in ${dev.languages.join(', ')}`;
}

// Class with generics
class Repository<T> {
  private items: T[] = [];

  add(item: T): void {
    this.items.push(item);
  }

  get(index: number): T | undefined {
    return this.items[index];
  }

  getAll(): readonly T[] {
    return this.items;
  }
}

// Usage
const developers = new Repository<Developer>();
developers.add({
  name: "Alice",
  occupation: "Software Developer",
  languages: ["TypeScript", "Python", "Go"],
  yearsExperience: 5
});

// Type guards
function isTechnology(obj: any): obj is Technology {
  return typeof obj === 'object' &&
         'name' in obj &&
         'category' in obj;
}

// Async/await with types
async function fetchTechnology(id: string): Promise<Technology> {
  const response = await fetch(`/api/technologies/${id}`);
  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }
  return response.json();
}
```

## Strict Mode

TypeScript's strict mode enables several type-checking flags:

- **noImplicitAny**: Error on expressions with implied 'any' type
- **strictNullChecks**: `null` and `undefined` are not assignable to other types
- **strictFunctionTypes**: Stronger checking of function types
- **strictBindCallApply**: Stricter checking of `bind`, `call`, and `apply`
- **strictPropertyInitialization**: Class properties must be initialized
- **noImplicitThis**: Error when `this` has implied 'any' type
- **alwaysStrict**: Parse in strict mode and emit "use strict"

## Migration Strategy

### Gradual Adoption
1. Rename `.js` files to `.ts`
2. Start with `any` types where needed
3. Add types incrementally
4. Enable stricter compiler options over time
5. Use `// @ts-ignore` or `// @ts-expect-error` sparingly

### Common Patterns
- Use `unknown` instead of `any` when possible
- Prefer interfaces for object types
- Use type aliases for unions and primitives
- Leverage type inference when obvious
- Document complex types with comments
