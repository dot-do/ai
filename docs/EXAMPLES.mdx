## Running Examples

### Prerequisites

- Node.js 18+
- pnpm package manager
- Valid .do API credentials

### Setup

```bash
# Install dependencies
pnpm install

# Set your API token
export DO_TOKEN=your-api-token

# Run any example
npx tsx examples/sdk/event-driven-order.ts
```

### Environment Variables

Create a `.env` file with your API credentials:

```bash
# .do Platform
DO_TOKEN=your-do-token

# External Services (for integration examples)
STRIPE_SECRET_KEY=sk_test_...
TWILIO_ACCOUNT_SID=AC...
TWILIO_AUTH_TOKEN=...
SENDGRID_API_KEY=SG...
SLACK_BOT_TOKEN=xoxb-...
OPENAI_API_KEY=sk-...
GOOGLE_MAPS_API_KEY=AIza...
```

## Example Patterns

### Event-Driven Workflows

Use `on` and `send` to build loosely-coupled systems:

```typescript
// Producer
on($.Order.created, async (order) => {
  await send($.Payment.process, order)
})

// Consumer
on($.Payment.process, async (payment) => {
  // Handle payment
})
```

### Scheduled Automation

Use `every` for recurring tasks:

```typescript
every($.Daily, async () => {
  // Generate daily report
})

every('0 */6 * * *', async () => {
  // Run every 6 hours
})
```

### AI Integration

Use `ai` for intelligent features:

```typescript
const content = await ai.generate({
  prompt: 'Create a blog post about AI',
  schema: $.BlogPosting,
})

const sentiment = await ai.generate({
  prompt: `Analyze sentiment: "${text}"`,
  schema: { sentiment: 'positive | negative | neutral' },
})
```

### Multi-Channel Notifications

Use `api` for external integrations:

```typescript
await Promise.all([
  api.sendgrid.send({ to: email, subject: 'Alert' }),
  api.twilio.messages.create({ to: phone, body: 'Alert' }),
  api.slack.chat.postMessage({ channel: '#alerts', text: 'Alert' }),
])
```

## Best Practices

### 1. Error Handling

Always wrap critical operations:

```typescript
on($.Payment.process, async (payment) => {
  try {
    await processPayment(payment)
    await send($.Payment.succeeded, payment)
  } catch (error) {
    await send($.Payment.failed, { ...payment, error: error.message })
  }
})
```

### 2. Idempotency

Ensure handlers can be safely retried:

```typescript
on($.Order.created, async (order) => {
  const existing = await db.get($.Invoice, `invoice-${order.$id}`)
  if (existing) return // Already processed

  await createInvoice(order)
})
```

### 3. Monitoring

Add logging and metrics:

```typescript
on($.Order.created, async (order) => {
  console.log(`Processing order ${order.$id}`)

  try {
    await processOrder(order)
    await send($.Metrics.increment, { metric: 'orders.processed' })
  } catch (error) {
    await send($.Metrics.increment, { metric: 'orders.failed' })
    throw error
  }
})
```

### 4. Testing

Write testable event handlers:

```typescript
export async function handleOrderCreated(order: Order) {
  // Business logic here
}

// Register handler
on($.Order.created, handleOrderCreated)

// Test handler directly
test('handles order creation', async () => {
  const order = createMockOrder()
  await handleOrderCreated(order)
  // Assert expected outcomes
})
```
