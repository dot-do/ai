# .do Platform Documentation

Welcome to the `.do` platform documentation. Learn how to build **autonomous Business-as-Code** with **AI-delivered Services-as-Software** using semantic, AI-native primitives.

## What is .do?

The `.do` platform enables developers to build applications using semantic `$.Subject.predicate.Object` patterns that both humans and AI can understand. Instead of writing imperative "how-to" code, you write declarative "what-it-means" code.

```typescript
import { $, ai, db, on } from '@dotdo/sdk.do'

// Create a business entity
const business = await db.create($.Business, {
  name: 'Acme Corp',
  description: 'We make great products',
})

// Listen for events
on($.Order.created, async (order) => {
  const email = await ai.generate('Write a thank you email', { context: order })
  console.log(email)
})
```

## Key Features

- **Semantic-First**: Built on `$.Subject.predicate.Object` patterns from RDF and Schema.org
- **AI-Native**: AI understands your code and can autonomously execute business logic
- **Type-Safe**: Full TypeScript support with autocomplete for 800+ Schema.org types
- **Event-Driven**: `on`, `send`, and `every` for reactive workflows
- **Zero Config**: Database schema, API endpoints, and AI prompts inferred from semantic types
