# Linear Issue Tracking & Project Management

Linear is a modern issue tracking and project management tool built for high-performance software teams. It provides fast, focused workflows for managing issues, projects, cycles, and team collaboration.

## Features

- **Fast Issue Tracking**: Streamlined issue creation and management with keyboard shortcuts
- **Projects & Roadmaps**: Track progress toward goals with projects and milestones
- **Sprint Planning**: Plan work in time-boxed cycles with automatic scheduling
- **Team Collaboration**: Comments, mentions, and notifications for seamless teamwork
- **Workflow Automation**: Customizable workflows and states for your team's process
- **GraphQL API**: Powerful, flexible API built on GraphQL
- **Webhooks**: Real-time event notifications for integrations
- **Native Apps**: Desktop and mobile apps with offline support

## Quick Start

```typescript
import { LinearClient } from '@dotdo/sdk.do/integrations/linear'

const linear = new LinearClient({
  apiKey: process.env.LINEAR_API_KEY,
})

// Create an issue
const issue = await linear.createIssue({
  teamId: 'team_abc123',
  title: 'Bug: Application crashes on startup',
  description: 'Detailed description...',
  priority: 1, // Urgent
  labelIds: ['label_bug', 'label_critical'],
})

// Search issues
const { nodes: issues } = await linear.searchIssues({
  filter: {
    team: { id: { eq: 'team_abc123' } },
    state: { type: { eq: 'started' } },
  },
  first: 50,
})
```

### OAuth 2.0

For public applications requiring user authorization:

1. Create an OAuth application in Linear settings
2. Implement OAuth 2.0 authorization code flow:

```typescript
// Step 1: Redirect user to authorization URL
const authUrl = `https://linear.app/oauth/authorize?client_id=${clientId}&redirect_uri=${redirectUri}&scope=read,write&response_type=code`

// Step 2: Exchange authorization code for access token
const tokenResponse = await fetch('https://api.linear.app/oauth/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    grant_type: 'authorization_code',
    client_id: clientId,
    client_secret: clientSecret,
    redirect_uri: redirectUri,
    code: authorizationCode,
  }),
})

const { access_token } = await tokenResponse.json()

// Step 3: Use access token with Linear client
const linear = new LinearClient({
  apiKey: access_token,
})
```

**OAuth Scopes:**

- `read` - Read access to all resources
- `write` - Create, update, and delete resources
- `admin` - Administrative access (manage team settings, users)

## Webhooks

### Webhook Verification

Always verify webhook signatures to ensure authenticity:

```typescript
import { LinearWebhooks } from '@dotdo/sdk.do/integrations/linear'

// In your webhook handler (e.g., Cloudflare Worker)
export default {
  async fetch(request: Request, env: Env) {
    const signature = request.headers.get('linear-signature')
    const payload = await request.text()

    try {
      // Verify and parse webhook
      const event = LinearWebhooks.verifyAndParse(payload, signature, env.LINEAR_WEBHOOK_SECRET)

      // Handle event
      console.log('Event:', event.type, event.action)
      console.log('Data:', event.data)

      return new Response('OK', { status: 200 })
    } catch (error) {
      console.error('Webhook error:', error)
      return new Response('Webhook Error', { status: 400 })
    }
  },
}
```

## Rate Limits

Linear enforces rate limits:

- **100 requests/second** (burst)
- **1,000 requests/hour**

Always implement retry logic with exponential backoff for production systems.

## Best Practices

1. **Use OAuth for public apps** - Personal API keys are for personal use only
2. **Always verify webhook signatures** - Prevent unauthorized webhook calls
3. **Implement retry logic** - Handle rate limits and transient failures gracefully
4. **Use pagination** - Don't fetch all data at once for large result sets
5. **Cache workflow states and labels** - These rarely change, reduce API calls
6. **Filter server-side** - Use GraphQL filters instead of client-side filtering
7. **Handle errors properly** - Check error types and implement appropriate handling
8. **Store API keys securely** - Never expose API keys in client-side code
