# Intercom Integration

Intercom is a customer messaging platform that enables businesses to communicate with customers through live chat, email, and in-app messages.

## Features

- **Contacts**: Manage customer contacts and profiles
- **Conversations**: Handle customer conversations and replies
- **Messages**: Send targeted messages to customers
- **Tags**: Organize contacts and conversations with tags
- **Articles**: Manage help center articles
- **Webhooks**: Real-time event notifications
- **API Key Auth**: Simple authentication

## Quick Start

```typescript
import { IntercomClient } from '@dotdo/sdk.do/integrations/intercom'

const intercom = new IntercomClient({
  apiKey: process.env.INTERCOM_ACCESS_TOKEN,
})

// Create a contact
const contact = await intercom.contact.create({
  email: 'user@example.com',
  name: 'John Doe',
})

// Send a message
const message = await intercom.message.create({
  message_type: 'in_app',
  from: { type: 'admin', id: 'admin-id' },
  to: { type: 'user', id: contact.id },
  body: 'Welcome to our platform!',
})
```

## Authentication

Intercom uses API token authentication:

```bash
export INTERCOM_ACCESS_TOKEN=your_access_token
```

## Resources

### Contacts

Manage customer contacts:

```typescript
// Create contact
const contact = await intercom.contact.create({
  email: 'user@example.com',
  name: 'Jane Smith',
  custom_attributes: {
    plan: 'pro',
    signup_date: '2025-10-12',
  },
})

// Update contact
await intercom.contact.update({
  id: contact.id,
  custom_attributes: {
    plan: 'enterprise',
  },
})

// List contacts
const contacts = await intercom.contact.list({ per_page: 50 })
```

### Conversations

Handle customer conversations:

```typescript
// Create conversation
const conversation = await intercom.conversation.create({
  from: { type: 'user', id: 'user-id' },
  body: 'I need help with...',
})

// Reply to conversation
await intercom.conversation.reply({
  id: conversation.id,
  message_type: 'comment',
  body: 'Happy to help!',
})

// List conversations
const conversations = await intercom.conversation.list()
```

### Messages

Send messages to customers:

```typescript
// Send in-app message
const message = await intercom.message.create({
  message_type: 'in_app',
  from: { type: 'admin', id: 'admin-id' },
  to: { type: 'user', email: 'user@example.com' },
  body: 'Check out our new features!',
})

// Send email
const email = await intercom.message.create({
  message_type: 'email',
  from: { type: 'admin', id: 'admin-id' },
  to: { type: 'user', email: 'user@example.com' },
  subject: 'Important Update',
  body: 'We have important news...',
})
```

### Tags

Organize with tags:

```typescript
// Create tag
const tag = await intercom.tag.create({ name: 'vip' })

// List tags
const tags = await intercom.tag.list()

// Delete tag
await intercom.tag.delete({ id: tag.id })
```

### Articles

Manage help center articles:

```typescript
// Create article
const article = await intercom.article.create({
  title: 'Getting Started Guide',
  body: '<h1>Welcome</h1><p>Getting started is easy...</p>',
  author_id: 'admin-id',
})

// Update article
await intercom.article.update({
  id: article.id,
  body: '<h1>Updated Guide</h1>',
})

// List articles
const articles = await intercom.article.list()
```

## Webhooks

Intercom supports webhooks for real-time notifications:

```typescript
import { IntercomWebhookHandler, WebhookEventRouter } from '@dotdo/sdk.do/integrations/intercom'

const handler = new IntercomWebhookHandler(process.env.INTERCOM_WEBHOOK_SECRET)
const event = await handler.handleRequest(request)

const router = new WebhookEventRouter()
router.onConversationCreated(async (event) => {
  console.log('New conversation:', event.data)
})

await router.route(event)
```

## Best Practices

1. **Rate Limits**: Intercom enforces rate limits - implement exponential backoff
2. **Contact Deduplication**: Use email or external_id to avoid duplicate contacts
3. **Custom Attributes**: Use custom attributes for flexible data storage
4. **Webhooks**: Verify webhook signatures for security
5. **Pagination**: Use cursor-based pagination for large datasets
6. **Message Types**: Choose appropriate message type (in_app vs email)
