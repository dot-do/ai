# Sentry Integration

Sentry is an error tracking and performance monitoring platform for developers.

## Features

- **Events**: Error and exception tracking
- **Issues**: Grouped error events
- **Projects**: Project organization
- **Releases**: Release tracking and deploys
- **Performance**: APM and transaction monitoring
- **Webhooks**: Real-time event notifications

## Quick Start

```typescript
import { SentryClient } from '@dotdo/sdk.do/integrations/sentry'

const sentry = new SentryClient({
  authToken: process.env.SENTRY_AUTH_TOKEN,
  organizationSlug: 'my-org',
})

// Get an issue
const issue = await sentry.issue.get({
  organization_slug: 'my-org',
  issue_id: '123456',
})

// Create a release
const release = await sentry.release.create({
  organization_slug: 'my-org',
  version: 'v2.0.0',
  projects: ['frontend', 'backend'],
})
```

## Authentication

Sentry uses API token authentication:

```bash
export SENTRY_AUTH_TOKEN=your_auth_token
export SENTRY_ORG_SLUG=your_organization_slug
```

## Resources

### Events

Retrieve error events:

```typescript
// Get event
const event = await sentry.event.get({
  organization_slug: 'my-org',
  event_id: 'abc123def456',
})

// List events
const events = await sentry.event.list({
  organization_slug: 'my-org',
  query: 'is:unresolved',
})
```

### Issues

Manage grouped errors:

```typescript
// Get issue
const issue = await sentry.issue.get({
  organization_slug: 'my-org',
  issue_id: '789',
})

// Update issue (resolve)
await sentry.issue.update({
  organization_slug: 'my-org',
  issue_id: '789',
  status: 'resolved',
})

// List issues
const issues = await sentry.issue.list({
  organization_slug: 'my-org',
  query: 'is:unresolved level:error',
})

// Delete issue
await sentry.issue.delete({
  organization_slug: 'my-org',
  issue_id: '789',
})
```

### Projects

Manage Sentry projects:

```typescript
// Create project
const project = await sentry.project.create({
  organization_slug: 'my-org',
  team_slug: 'my-team',
  name: 'New Service',
  platform: 'node',
})

// Get project
const project = await sentry.project.get({
  organization_slug: 'my-org',
  project_slug: 'new-service',
})

// Update project
await sentry.project.update({
  organization_slug: 'my-org',
  project_slug: 'new-service',
  name: 'Updated Service Name',
})

// List projects
const projects = await sentry.project.list({
  organization_slug: 'my-org',
})

// Delete project
await sentry.project.delete({
  organization_slug: 'my-org',
  project_slug: 'new-service',
})
```

### Releases

Track releases and deploys:

```typescript
// Create release
const release = await sentry.release.create({
  organization_slug: 'my-org',
  version: 'v1.5.0',
  projects: ['frontend', 'backend'],

  organization_slug: 'my-org',

  organization_slug: 'my-org',
})

// Delete release
await sentry.release.delete({
  organization_slug: 'my-org',
  version: 'v1.5.0',
})
```

## Webhooks

Sentry supports webhooks with HMAC-SHA256 signature verification:

```typescript
import { SentryWebhookHandler, WebhookEventRouter } from '@dotdo/sdk.do/integrations/sentry'

const handler = new SentryWebhookHandler(process.env.SENTRY_WEBHOOK_SECRET)
const event = await handler.handleRequest(request)

const router = new WebhookEventRouter()
router.onErrorCreated(async (event) => {
  console.log('New error:', event.data)
})

router.onIssueResolved(async (event) => {
  console.log('Issue resolved:', event.data)
})

await router.route(event)
```

      // Retry logic
    }
  }

## Best Practices

1. **Organization Slug**: All operations scoped to organization
2. **Issue Management**: Use status updates to manage workflow
3. **Release Tracking**: Associate commits with releases
4. **Performance**: Use performance monitoring for transaction tracking
5. **Webhooks**: Verify signatures for security
6. **Rate Limits**: Respect API rate limits
7. **Search Queries**: Use Sentry search syntax for filtering
