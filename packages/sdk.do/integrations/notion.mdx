# Notion Workspace Integration

Notion is an all-in-one workspace that combines notes, wikis, databases, and project management. It provides a flexible platform for organizing information, collaborating with teams, and building custom workflows.

## Features

- **Pages & Content**: Rich text editing with 50+ content block types (paragraphs, headings, lists, tables, embeds)
- **Databases**: Structured data with 15+ property types (text, number, select, date, relation, etc.)
- **Relations & Rollups**: Link databases and aggregate data across pages
- **Real-time Collaboration**: Multi-user editing with comments and mentions
- **Templates**: Reusable page and database templates
- **API Access**: Full programmatic access to all workspace content
- **Webhooks**: Real-time event notifications for changes
- **OAuth 2.0**: Secure user authentication and authorization

## Quick Start

```typescript
import { NotionClient } from '@dotdo/sdk.do/integrations/notion'

const notion = new NotionClient({
  auth: process.env.NOTION_TOKEN,
})

// Create a page
const page = await notion.createPage({
  parent: { page_id: 'parent-page-id' },
  properties: {
    title: {
      title: [
        {
          text: {
            content: 'My New Page',
          },
        },
      ],
    },
  },
  children: [
    {
      type: 'paragraph',
      paragraph: {
        rich_text: [
          {
            text: {
              content: 'Hello from the Notion API!',
            },
          },
        ],
      },
    },
  ],
})

// Query a database
const results = await notion.queryDatabase('database-id', {
  filter: {
    property: 'Status',
    select: {
      equals: 'In Progress',
    },
  },
  sorts: [
    {
      property: 'Priority',
      direction: 'descending',
    },
  ],
})
```

## Authentication

Notion supports two authentication methods:

### 1. Internal Integration (Server-to-Server)

Create an internal integration in your workspace:

1. Go to https://www.notion.so/my-integrations
2. Click "New integration"
3. Select capabilities: Read content, Update content, Insert content, Comment
4. Copy the "Internal Integration Token"
5. Share pages/databases with your integration

```typescript
const notion = new NotionClient({
  auth: process.env.NOTION_TOKEN, // Internal Integration Token
})
```

### 2. OAuth 2.0 (User Authorization)

For public integrations that access user workspaces:

```typescript
// Step 1: Redirect user to Notion authorization
const authUrl = `https://api.notion.com/v1/oauth/authorize?client_id=${CLIENT_ID}&response_type=code&redirect_uri=${REDIRECT_URI}`

// Step 2: Exchange authorization code for access token
const response = await fetch('https://api.notion.com/v1/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Basic ${Buffer.from(`${CLIENT_ID}:${CLIENT_SECRET}`).toString('base64')}`,
  },
  body: JSON.stringify({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: REDIRECT_URI,
  }),
})

const { access_token } = await response.json()

// Step 3: Use access token
const notion = new NotionClient({
  auth: access_token,
})
```

Pages are the fundamental building blocks of Notion. They can exist standalone or within databases.

```typescript

// Get page by ID
const retrieved = await notion.getPage(page.id)

// Update page properties
await notion.updatePage(page.id, {
  properties: {
    title: {
      title: [{ text: { content: 'Updated Title' } }],
    },
  },
  icon: {
    type: 'emoji',
    emoji: '📝',
  },
  cover: {
    type: 'external',
    external: { url: 'https://example.com/cover.jpg' },
  },

// Archive page

// Search pages

  sort: {
    direction: 'descending',
    timestamp: 'last_edited_time',
  },
})
```

## Webhooks

Notion sends webhook events when pages, databases, blocks, or comments are created, updated, or deleted.

### Setting Up Webhooks

1. Go to your integration settings
2. Add webhook URL endpoint
3. Copy webhook secret for signature verification
4. Subscribe to event types

### Webhook Verification

```typescript
import { NotionWebhooks, EventRouter } from '@dotdo/sdk.do/integrations/notion'

// Verify webhook signature
const isValid = NotionWebhooks.verifySignature(
  requestBody, // Raw body string
  request.headers['notion-signature'],
  process.env.NOTION_WEBHOOK_SECRET
)

if (!isValid) {
  return new Response('Invalid signature', { status: 401 })
}

// Parse event
const event = NotionWebhooks.parseEvent(requestBody)
```

### Event Handling

```typescript
const router = new EventRouter()

// Handle specific events
router.on('page.created', async (event, page) => {
  console.log('New page created:', page.id)
  // Process new page
})

router.on('page.updated', async (event, page) => {
  console.log('Page updated:', page.id)
  // Sync changes
})

router.on('database.updated', async (event, database) => {
  console.log('Database schema changed:', database.id)
  // Update local schema
})

// Handle all page events with wildcards
router.on('page.*', async (event, page) => {
  console.log('Page event:', event.type, page.id)
})

// Handle all events
router.on('*', async (event, data) => {
  console.log('Event received:', event.type)
})

// Process webhook
await router.handle(event)
```

### Available Events

- **Page Events**: `page.created`, `page.updated`, `page.deleted`
- **Database Events**: `database.created`, `database.updated`, `database.deleted`
- **Block Events**: `block.created`, `block.updated`, `block.deleted`
- **Comment Events**: `comment.created`

## Error Handling

```typescript
import { NotionIntegrationError, NotionErrorCode } from '@dotdo/sdk.do/integrations/notion'

try {
  await notion.getPage('invalid-page-id')
} catch (error) {
  if (error instanceof NotionIntegrationError) {
    console.error('Error:', error.code, error.message)

    switch (error.code) {
      case NotionErrorCode.AUTHENTICATION_FAILED:
        // Invalid or expired token
        break

      case NotionErrorCode.PERMISSION_DENIED:
        // Integration lacks required capabilities
        // Or page/database not shared with integration
        break

      case NotionErrorCode.OBJECT_NOT_FOUND:
        // Page, database, or block doesn't exist
        break

      case NotionErrorCode.RATE_LIMIT_EXCEEDED:
        // Exceeded 3 requests per second
        if (error.isRetriable()) {
          // Wait and retry
          await new Promise((resolve) => setTimeout(resolve, 1000))
        }
        break

      case NotionErrorCode.VALIDATION_ERROR:
        // Invalid request parameters
        break
    }
  }
}
```

## Rate Limits

Notion enforces the following rate limits:

- **3 requests per second** per integration
- **Burst allowance**: Up to 10 requests in quick succession
- **Rate limit headers**: Check `Retry-After` header when rate limited

Best practices:

1. Implement exponential backoff for retries
2. Batch operations where possible
3. Cache frequently accessed data
4. Use webhooks instead of polling

## Best Practices

1. **Share with Integration**: Always share pages/databases with your integration before accessing
2. **Use Webhooks**: Prefer webhooks over polling for real-time updates
3. **Batch Operations**: Group multiple block appends into single requests
4. **Error Handling**: Implement comprehensive error handling with retries for transient errors
5. **Property Types**: Validate property types before creating/updating pages
6. **Pagination**: Always handle pagination for list operations (100 items per page)
7. **Rich Text**: Use rich text arrays for formatted content with annotations
8. **Archive Instead of Delete**: Notion archives pages/databases instead of deleting them
9. **API Version**: Pin to a specific API version for stability
10. **Rate Limiting**: Respect rate limits and implement exponential backoff

### Sync Content from External System

// Sync projects from external system
for (const project of externalProjects) {

### Generate Report from Database

```typescript
// Query tasks by status
const tasks = await notion.queryDatabase('tasks-database-id', {
  filter: {
    property: 'Status',
    select: { equals: 'Done' },
  },
})

// Create report page
const report = await notion.createPage({
  parent: { page_id: 'reports-page-id' },
  properties: {
    title: {
      title: [{ text: { content: 'Weekly Report' } }],
    },
  },
})

// Add task summary
const blocks = tasks.results.map((task) => ({
  type: 'bulleted_list_item',
  bulleted_list_item: {
    rich_text: [
      {
        text: {
          content: task.properties.Task.title[0]?.text.content || 'Untitled',
        },
      },
    ],
  },
}))

await notion.appendBlockChildren(report.id, { children: blocks })
```
