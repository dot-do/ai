# Jira Integration

Jira is an agile project management and issue tracking platform for software teams.

## Features

- **Issues**: Issue tracking and management
- **Projects**: Project organization and configuration
- **Sprints**: Agile sprint planning and execution
- **Boards**: Scrum and Kanban boards
- **OAuth 2.0**: Secure authentication
- **Webhooks**: Real-time event notifications with HMAC

## Quick Start

```typescript
import { JiraClient } from '@dotdo/sdk.do/integrations/jira'

const jira = new JiraClient({
  host: 'your-instance.atlassian.net',
  email: process.env.JIRA_EMAIL,
  apiToken: process.env.JIRA_API_TOKEN,
})

// Create an issue
const issue = await jira.issue.create({
  fields: {
    project: { key: 'PROJ' },
    summary: 'Bug: Login fails',
    issuetype: { name: 'Bug' },
    description: 'Users cannot log in',
  },
})

// Create a sprint
const sprint = await jira.sprint.create({
  name: 'Sprint 1',
  originBoardId: 123,
  startDate: '2025-10-13',
  endDate: '2025-10-27',
})
```

## Authentication

Jira uses OAuth 2.0 or API tokens:

```bash
export JIRA_EMAIL=your_email@example.com
export JIRA_API_TOKEN=your_api_token
export JIRA_HOST=your-instance.atlassian.net
```

## Resources

### Issues

Track and manage issues:

```typescript
// Create issue
const issue = await jira.issue.create({
  fields: {
    project: { key: 'PROJ' },
    summary: 'Implement feature X',
    issuetype: { name: 'Story' },
    priority: { name: 'High' },
  },
})

// Get issue
const issue = await jira.issue.get({
  issue_id_or_key: 'PROJ-123',
})

// Update issue
await jira.issue.update({
  issue_id_or_key: 'PROJ-123',
  fields: {
    summary: 'Updated: Implement feature X',
  },
})

// Search issues with JQL
const issues = await jira.issue.list({
  jql: 'project = PROJ AND status = "In Progress"',
  fields: ['summary', 'status', 'assignee'],
})

// Delete issue
await jira.issue.delete({
  issue_id_or_key: 'PROJ-123',
})
```

### Projects

Manage projects:

```typescript
// Create project
const project = await jira.project.create({
  key: 'NEWPROJ',
  name: 'New Project',

  name: 'Updated Project Name',

// List projects

// Delete project
await jira.project.delete({
  project_id_or_key: 'NEWPROJ',
})
```

Manage agile sprints:

```typescript

// Get sprint

  sprint_id: 100,
})

// Update sprint
await jira.sprint.update({
  sprint_id: 100,
  name: 'Sprint 10 Extended',
  state: 'active',
})

// List sprints for board
const sprints = await jira.sprint.list({
  board_id: 42,

// Delete sprint
await jira.sprint.delete({
  sprint_id: 100,
})
```

## Webhooks

Jira supports webhooks with HMAC-SHA256 signature verification:

```typescript
import { JiraWebhookHandler, WebhookEventRouter } from '@dotdo/sdk.do/integrations/jira'

const handler = new JiraWebhookHandler(process.env.JIRA_WEBHOOK_SECRET)
const event = await handler.handleRequest(request)

const router = new WebhookEventRouter()
router.onJiraIssueCreated(async (event) => {
  console.log('Issue created:', event.data)
})

router.onJiraIssueUpdated(async (event) => {
  console.log('Issue updated:', event.data)
})

router.onSprintStarted(async (event) => {
  console.log('Sprint started:', event.data)
})

await router.route(event)
```

## Best Practices

1. **JQL**: Master Jira Query Language for powerful issue searches
2. **Issue Keys**: Use project-key format (e.g., PROJ-123)
3. **Fields**: Always specify required fields for issue creation
4. **Pagination**: Use startAt and maxResults for large datasets
5. **Webhooks**: Verify HMAC signatures for security
6. **Rate Limits**: Jira Cloud has per-app rate limits
7. **Agile API**: Use /rest/agile/1.0 for sprints and boards
