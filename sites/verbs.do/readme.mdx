# verbs.do

**Semantic action verbs and operations for Business-as-Code applications**

verbs.do provides a comprehensive, AI-native vocabulary of action verbs and operations using GraphDL's `$.Subject.predicate.Object` semantic patterns. It enables developers and AI agents to express business operations with clarity and type safety.

## Overview

In Business-as-Code, **verbs** represent actions and operations - what you can **do** with entities. verbs.do provides:

- **CRUD Operations** - create, read, update, delete patterns
- **Semantic Predicates** - worksFor, owns, manages, produces
- **Action Types** - Schema.org Action hierarchy (ConsumeAction, TradeAction, etc.)
- **GraphDL Patterns** - `$.Subject.verb.Object` semantic relationships
- **Type-Safe Operations** - TypeScript definitions for all verbs
- **Business Workflows** - Common business operation patterns

## Quick Start

Install the SDK:

```bash
pnpm add sdk.do
```

Use semantic verbs in your code:

```typescript
import $, { db } from 'sdk.do'

// CRUD operations
const person = await $.create($.Person, { name: 'Jane' })
const people = await $.list($.Person)
await $.update(person, { jobTitle: 'CTO' })
await $.delete(person)

// Semantic predicates
await $.assign(person, organization)
await $.notify(person, { message: 'Welcome!' })
await $.approve(order, person)

// GraphDL relationships
await db.relate(person, $.worksFor, organization)
await db.relate(organization, $.owns, brand)
await db.relate(brand, $.manufactures, product)

// Action types
const action = await db.create($.SendAction, {
  $type: 'SendAction',
  agent: person,
  recipient: otherPerson,
  object: message,
})
```

## Core Concepts

### What Are Verbs?

In Business-as-Code, **verbs** are operations that express what you can do:

- **CRUD Verbs**: create, read, update, delete, list, count
- **Relationship Verbs**: assign, relate, link, connect, associate
- **Business Verbs**: approve, reject, process, fulfill, cancel
- **Communication Verbs**: send, notify, message, email, call
- **State Verbs**: activate, deactivate, enable, disable, archive
- **GraphDL Predicates**: worksFor, owns, manages, produces, sells

### The $.Subject.verb.Object Pattern

verbs.do uses semantic triple patterns:

```typescript
// Pattern: $.Subject.verb.Object
$.Person.worksFor.Organization
$.Organization.owns.Brand
$.Brand.manufactures.Product
$.Person.manages.Team

// Usage in relationships
await db.relate(person, $.worksFor, organization)
const employer = await db.related(person, $.worksFor, $.Organization)

// Usage in queries
const employees = await db.related(organization, $.employs, $.Person)
const products = await db.related(brand, $.manufactures, $.Product)
```

## CRUD Operations

### Create

Create new entities:

```typescript
// Simple create
const person = await $.create($.Person, {
  $type: 'Person',
  name: 'Jane Smith',
  email: 'jane@example.com',
})

// Create with relationships
const order = await $.create($.Order, {
  $type: 'Order',
  orderNumber: 'ORD-001',
  customer: { $id: person.$id, $type: 'Person' },
  orderStatus: 'Pending',
})

// Batch create
const people = await $.createMany($.Person, [{ name: 'Alice' }, { name: 'Bob' }, { name: 'Carol' }])
```

### Read

Read and query entities:

```typescript
// Get by ID
const person = await $.get($.Person, 'person-123')

// List all
const people = await $.list($.Person)

// List with filters
const engineers = await $.list($.Person, {
  where: { jobTitle: { $contains: 'Engineer' } },
  limit: 10,
})

// Count
const personCount = await $.count($.Person)

// Exists check
const exists = await $.exists($.Person, 'person-123')
```

### Update

Update entity properties:

```typescript
// Update single property
await $.update(person, { jobTitle: 'Senior Engineer' })

// Update multiple properties
await $.update(person, {
  jobTitle: 'Lead Engineer',
  email: 'jane.smith@newcompany.com',
})

// Partial update
await $.patch(person, { telephone: '+1-555-0199' })

// Batch update
await $.updateMany($.Person, {
  where: { department: 'Engineering' },
  data: { status: 'Active' },
})
```

### Delete

Remove entities:

```typescript
// Delete single
await $.delete(person)

// Delete by ID
await $.delete($.Person, 'person-123')

// Batch delete
await $.deleteMany($.Person, {
  where: { status: 'Inactive' },
})

// Soft delete (archive)
await $.archive(person)
```

## Business Workflows

### Order Processing

```typescript
async function processOrder(orderId: string) {
  const order = await $.get($.Order, orderId)

  // Validate order
  await $.validate(order)

  // Process payment
  const payment = await $.processPayment(order)

  // Fulfill order
  await $.fulfill(order)

  // Ship order
  await $.ship(order, {
    carrier: 'UPS',
    trackingNumber: 'TRACK123',
  })

  // Update status
  await $.update(order, {
    orderStatus: 'Shipped',
    trackingNumber: 'TRACK123',
  })

  // Notify customer
  await $.notify(order.customer, {
    message: `Order ${order.orderNumber} has been shipped`,
  })
}
```

### Lead Qualification

```typescript
async function qualifyLead(leadId: string) {
  const lead = await $.get($.Lead, leadId)

  // Score lead
  const score = await $.score(lead)

  if (score > 70) {
    // Qualify lead
    await $.qualify(lead)

    // Assign to sales rep
    await $.assign(lead, {
      assignedTo: salesRep.$id,
    })

    // Notify sales rep
    await $.notify(salesRep, {
      message: `New qualified lead: ${lead.name}`,
    })

    // Convert to account
    if (lead.readyToConvert) {
      const account = await $.convert(lead)
      return account
    }
  }

  return lead
}
```

### Employee Onboarding

```typescript
async function onboardEmployee(employeeId: string) {
  const employee = await $.get($.Person, employeeId)
```

## Advanced Patterns

### Custom Verb Registration

```typescript
import { registry } from 'verbs.do/core'

// Register custom verb
registry.registerVerb({
  name: 'approve',
  description: 'Approve an entity for processing',
  params: {
    entity: 'Thing',
    approver: 'Person',
    comments: 'string',
  },
  returns: 'ApprovalResult',
})

// Use custom verb
await $.approve(order, {
  approver: manager,
  comments: 'Approved for processing',
})
```

### Verb Composition

```typescript
// Compose multiple verbs into workflow
async function processAndShipOrder(orderId: string) {
  const order = await $.get($.Order, orderId)

  // Compose operations
  await $.compose([() => $.validate(order), () => $.processPayment(order), () => $.fulfill(order), () => $.ship(order), () => $.notify(order.customer)])
}
```

### Conditional Verbs

```typescript
// Execute verb based on condition
await $.when(order.total > 1000, () => {
  return $.requireApproval(order, manager)
})

// Switch on verb
await $.switch(order.status, {
  Pending: () => $.process(order),
  Processing: () => $.fulfill(order),
  Fulfilled: () => $.ship(order),
  Shipped: () => $.complete(order),
})
```

## Best Practices

### 1. Use Semantic Verbs

```typescript
// ✓ Good - semantic verb
await $.assign(person, organization)

// ✗ Avoid - string-based
await performAction('assign', person, organization)
```

### 2. Compose Operations

```typescript
// ✓ Good - composed workflow
async function processOrder(order) {
  await $.validate(order)
  await $.processPayment(order)
  await $.fulfill(order)
  await $.notify(order.customer)
}

// ✗ Avoid - monolithic
async function processOrder(order) {
  // All logic in one function
}
```

### 3. Handle Errors

```typescript
// ✓ Good - error handling
try {
  await $.processPayment(order)
} catch (error) {
  await $.refund(order)
  await $.notify(customer, { message: 'Payment failed' })
}
```
