# bots.do

**Digital Worker Bot Framework for Business Automation**

bots.do is the bot framework for the `.do` platform, enabling you to build autonomous digital worker bots that automate tasks, execute workflows, and collaborate in multi-agent systems. Create bots with specific capabilities, tools, and behaviors that can work independently or as part of larger agent teams.

## What are Bots?

Bots are specialized autonomous entities that:

- **Execute**: Perform specific tasks with precision and consistency
- **Automate**: Handle repetitive workflows without human intervention
- **Integrate**: Connect with APIs, databases, and external systems
- **Collaborate**: Work with other bots and agents in coordinated systems
- **Scale**: Handle high-volume operations efficiently

Unlike general-purpose agents, bots are optimized for specific, well-defined tasks with predictable patterns and high-volume execution requirements.

## Quick Start

```bash
pnpm add sdk.do
```

```typescript
import $, { bot } from 'sdk.do'

// Define a simple bot
const emailBot = await bot.create({
  $type: 'Bot',
  name: 'Email Processor Bot',
  task: 'Process incoming emails',
  capabilities: ['email_parsing', 'classification', 'routing'],
  triggers: ['$.Email.received'],
  model: 'gpt-5-mini',
})

// Execute bot task
const result = await emailBot.execute({
  input: { email: emailData },
})
```

## Key Features

### Task-Specific Bots

Create bots optimized for specific tasks:

```typescript
const dataBot = await bot.create({
  $type: 'Bot',

  capabilities: ['data_extraction', 'data_transformation', 'data_loading'],
  sources: ['crm', 'database', 'api'],
  destinations: ['warehouse', 'analytics'],
  errorHandling: {
    retry: { attempts: 3, backoff: 'exponential' },
    alertOnFailure: true,
  },
})
```

### High-Volume Automation

Handle large-scale operations:

```typescript
const processingBot = await bot.create({
  $type: 'Bot',
  name: 'Invoice Processing Bot',
  task: 'Process invoices',
  throughput: {
    target: 1000, // invoices per hour
    concurrency: 10,
    batchSize: 100,
  },
  pipeline: ['extract_data', 'validate_fields', 'calculate_totals', 'store_record', 'send_notification'],
})

// Process batch
await processingBot.processBatch({
  invoices: invoiceList,
})
```

### Event-Driven Execution

Trigger bots from events:

```typescript
import { on } from 'sdk.do'

// Bot triggered by events
const notificationBot = await bot.create({
  $type: 'Bot',
  name: 'Notification Bot',
  task: 'Send notifications',
  channels: ['email', 'slack', 'sms'],
})

// Listen for events
await on($.Order.completed, async (order) => {
  await notificationBot.execute({
    action: 'send_confirmation',
    recipient: order.customer,
    data: order,
  })
})

await on($.Payment.failed, async (payment) => {
  await notificationBot.execute({
    action: 'send_alert',
    recipient: 'finance@company.com',
    data: payment,
  })
})
```

### Multi-Bot Orchestration

Coordinate multiple bots:

```typescript
const botTeam = await bot.createTeam({
  name: 'Order Processing Team',
  bots: [
    {
      id: 'validator',
      bot: validationBot,
      role: 'Validate order data',
    },
    {
      id: 'inventory',
      bot: inventoryBot,
      role: 'Check inventory availability',
    },
    {
      id: 'payment',
      bot: paymentBot,
      role: 'Process payment',
    },
    {
      id: 'fulfillment',
      bot: fulfillmentBot,
      role: 'Create fulfillment order',
    },
  ],
  workflow: {
    type: 'sequential',
    stages: [
      { bot: 'validator', continueOnSuccess: true },
      { bot: 'inventory', continueOnSuccess: true },
      { bot: 'payment', continueOnSuccess: true },
      { bot: 'fulfillment', continueOnSuccess: true },
    ],
    errorHandling: 'rollback',
  },
})

// Execute orchestrated workflow
const result = await botTeam.processOrder(orderData)
```

### Bot Intelligence Levels

Choose the right intelligence level:

```typescript
// Rule-based bot (fastest, most predictable)
const ruleBot = await bot.create({
  $type: 'Bot',
  name: 'Rule Bot',
  intelligence: 'rules',
  rules: [
    { if: 'amount > 10000', then: 'flag_for_review' },
    { if: 'customer.vip === true', then: 'priority_processing' },
    { if: 'region === "EU"', then: 'apply_gdpr_rules' }
  ]
})

// AI-powered bot (adaptive, context-aware)
const aiBot = await bot.create({
  $type: 'Bot',
  name: 'AI Bot',
  intelligence: 'ai',
  model: 'gpt-5-mini',
  context: 'Process customer inquiries with contextual understanding',
  capabilities: ['natural_language', 'sentiment_analysis', 'intent_classification']
})

// Hybrid bot (rules + AI)
const hybridBot = await bot.create({
  $type: 'Bot',
  name: 'Hybrid Bot',
  intelligence: 'hybrid',
  rules: [...], // Fast path for common cases
  model: 'gpt-5-mini', // AI for complex cases
  fallback: 'ai' // Use AI when rules don't match
})
```

## Bot Lifecycle

### 1. Define Bot

```typescript
const botDef = {
  $type: 'Bot',
  name: 'Invoice Bot',
  task: 'Process invoices',
  capabilities: ['pdf_parsing', 'data_extraction', 'validation'],
  model: 'gpt-5-mini',
}
```

### 2. Create Bot Instance

```typescript
const bot = await bot.create(botDef)
```

### 3. Configure Bot Behavior

```typescript
await bot.configure({
  concurrency: 5,
  batchSize: 100,
  retryPolicy: { attempts: 3, backoff: 'exponential' },
  monitoring: { logLevel: 'info', metricsEnabled: true },
})
```

### 4. Deploy Bot

```typescript
await bot.deploy({
  environment: 'production',
  scaling: {
    min: 1,
    max: 10,
    metric: 'queue_depth',
    threshold: 100,
  },
})
```

### 5. Monitor Bot

```typescript
const status = await bot.getStatus()
// {
//   state: 'running',
//   tasksProcessed: 15420,
//   successRate: 0.987,
//   avgProcessingTime: 234,
//   lastError: null
// }
```
