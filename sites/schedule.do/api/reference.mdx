# schedule.do API Reference

Complete API documentation for schedule.do functions, types, and interfaces.

## Functions

### schedule()

Main function for task scheduling.

**Signature:**

```typescript
async function schedule<T = any>(data: T | ScheduleInput, options?: ScheduleOptions): Promise<ScheduleResult<T>>
```

**Parameters:**

- `data` - Input data to process
- `options` - Optional configuration options

**Returns:** Promise resolving to result object

**Example:**

```typescript
const result = await schedule({
  data: 'input',
  options: { format: 'json' },
})
```

### schedule.create()

Create a reusable instance.

**Signature:**

```typescript
function create(config: ScheduleConfig): ScheduleInstance
```

**Parameters:**

- `config` - Instance configuration

**Returns:** Instance object

**Example:**

```typescript
const instance = schedule.create({
  timeout: 30000,
  retries: 3,
})

const result = await instance.process(data)
```

### schedule.batch()

Process multiple items.

**Signature:**

```typescript
async function batch<T>(items: T[], options?: BatchOptions): Promise<ScheduleResult<T>[]>
```

**Parameters:**

- `items` - Array of items to process
- `options` - Optional batch options

**Returns:** Promise resolving to array of results

**Example:**

```typescript
const results = await schedule.batch([item1, item2, item3])
```

### schedule.stream()

Process data as a stream.

**Signature:**

```typescript
async function* stream<T>(
  data: AsyncIterable<T>,
  options?: StreamOptions
): AsyncGenerator<ScheduleResult<T>>
```

**Parameters:**

- `data` - Async iterable data source
- `options` - Optional stream options

**Returns:** Async generator yielding results

**Example:**

```typescript
for await (const result of schedule.stream(dataStream)) {
  console.log('Result:', result)
}
```

## Types

### ScheduleOptions

Configuration options for schedule().

```typescript
interface ScheduleOptions {
  // Format options
  format?: 'json' | 'xml' | 'text'

  // Validation
  validate?: boolean
  strict?: boolean

  // Performance
  timeout?: number
  cache?: boolean
  cacheTTL?: number

  // Error handling
  throwOnError?: boolean
  retries?: number
}
```

### ScheduleResult

Result object returned by schedule().

```typescript
interface ScheduleResult<T = any> {
  // Success indicator
  success: boolean

  // Result data
  data?: T

  // Error information
  error?: string
  code?: string

  // Metadata
  metadata?: {
    duration: number
    timestamp: Date
    cached: boolean
  }
}
```

### ScheduleConfig

Configuration for creating instances.

```typescript
interface ScheduleConfig {
  // API configuration
  apiKey?: string
  endpoint?: string

  // Timeouts
  timeout?: number
  retries?: number

  // Defaults
  defaultOptions?: ScheduleOptions
}
```

### ScheduleInstance

Instance created by schedule.create().

```typescript
interface ScheduleInstance {
  // Process method
  process<T>(data: T, options?: ScheduleOptions): Promise<ScheduleResult<T>>

  // Batch method
  batch<T>(items: T[], options?: BatchOptions): Promise<ScheduleResult<T>[]>

  // Close instance
  close(): Promise<void>

  // Get metrics
  getMetrics(): Metrics
}
```

### ScheduleError

Custom error class.

```typescript
class ScheduleError extends Error {
  code: string
  details?: any

  constructor(message: string, code: string, details?: any)
}
```

```typescript
const DEFAULT_OPTIONS: ScheduleOptions = {

  validate: true,
  timeout: 30000,
  retries: 3,
}
```

## Events

### Instance Events

Instances emit events for monitoring:

```typescript
instance.on('start', (data) => {
  console.log('Processing started:', data)
})

instance.on('complete', (result) => {
  console.log('Processing completed:', result)
})

instance.on('error', (error) => {
  console.error('Processing error:', error)
})
```
