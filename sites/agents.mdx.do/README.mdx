## Quick Start

### 1. Create Your Configuration

Create an `agents.md` file:

```markdown
# MyAgent

Description of your agent

## Instructions

Core instructions for the agent

## Tools

- tool1: Description
- tool2: Description

## Rules

- Rule 1
- Rule 2
```

### 2. Convert to Any Format

```bash
# Convert to Cursor IDE format
agents render agents.md --to cursor > .cursorrules

# Convert to Claude format
agents render agents.md --to claude > CLAUDE.md

# Convert to JSON
agents render agents.md --to json > config.json
```

### 3. Validate & Deploy

```bash
# Validate your configuration
agents validate agents.md --strict

# Deploy to your AI assistant
git add CLAUDE.md .cursorrules
git commit -m "Update agent configuration"
```

**That's it!** Your agent is now configured for multiple AI assistants.

## API Overview

### Parsing

```typescript

// Auto-detect format

const config = parse(content, { format: 'cursor' })
```

### Rendering


## FAQ

### Q: Can I add custom format support?

**A:** Yes! The parser and renderer are extensible. See the [custom formats guide](https://docs.do/agents#custom-formats) for details.

### Q: Does this work with my existing configs?

**A:** Yes! Parse your existing `.cursorrules` or `CLAUDE.md` files and convert them to the universal format.

### Q: What about private/sensitive configurations?

**A:** Everything runs locally. Your configurations never leave your machine unless you explicitly push them to version control.

### Q: Can I use this in CI/CD?

**A:** Absolutely! Use the CLI for validation and conversion in your build pipeline.

```yaml
# .github/workflows/validate-agent.yml
- name: Validate agent config
  run: agents validate CLAUDE.md --strict
```

### Q: Which AI assistants are supported?

**A:** Currently: Claude AI, Cursor IDE, Windsurf, and generic JSON. More formats are being added regularly.

### Q: Is there a size limit for configurations?

**A:** No hard limits, but we recommend keeping configs under 50KB for optimal performance.
