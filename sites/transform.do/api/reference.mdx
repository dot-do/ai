# transform.do API Reference

Complete API documentation for transform.do functions, types, and interfaces.

## Functions

### transform()

Main function for data transformation.

**Signature:**

```typescript
async function transform<T = any>(data: T | TransformInput, options?: TransformOptions): Promise<TransformResult<T>>
```

**Parameters:**

- `data` - Input data to process
- `options` - Optional configuration options

**Returns:** Promise resolving to result object

**Example:**

```typescript
const result = await transform({
  data: 'input',
  options: { format: 'json' },
})
```

### transform.create()

Create a reusable instance.

**Signature:**

```typescript
function create(config: TransformConfig): TransformInstance
```

**Parameters:**

- `config` - Instance configuration

**Returns:** Instance object

**Example:**

```typescript
const instance = transform.create({
  timeout: 30000,
  retries: 3,
})

const result = await instance.process(data)
```

### transform.batch()

Process multiple items.

**Signature:**

```typescript
async function batch<T>(items: T[], options?: BatchOptions): Promise<TransformResult<T>[]>
```

**Parameters:**

- `items` - Array of items to process
- `options` - Optional batch options

**Returns:** Promise resolving to array of results

**Example:**

```typescript
const results = await transform.batch([item1, item2, item3])
```

### transform.stream()

Process data as a stream.

**Signature:**

```typescript
async function* stream<T>(
  data: AsyncIterable<T>,
  options?: StreamOptions
): AsyncGenerator<TransformResult<T>>
```

**Parameters:**

- `data` - Async iterable data source
- `options` - Optional stream options

**Returns:** Async generator yielding results

**Example:**

```typescript
for await (const result of transform.stream(dataStream)) {
  console.log('Result:', result)
}
```

## Types

### TransformOptions

Configuration options for transform().

```typescript
interface TransformOptions {
  // Format options
  format?: 'json' | 'xml' | 'text'

  // Validation
  validate?: boolean
  strict?: boolean

  // Performance
  timeout?: number
  cache?: boolean
  cacheTTL?: number

  // Error handling
  throwOnError?: boolean
  retries?: number
}
```

### TransformResult

Result object returned by transform().

```typescript
interface TransformResult<T = any> {
  // Success indicator
  success: boolean

  // Result data
  data?: T

  // Error information
  error?: string
  code?: string

  // Metadata
  metadata?: {
    duration: number
    timestamp: Date
    cached: boolean
  }
}
```

### TransformConfig

Configuration for creating instances.

```typescript
interface TransformConfig {
  // API configuration
  apiKey?: string
  endpoint?: string

  // Timeouts
  timeout?: number
  retries?: number

  // Defaults
  defaultOptions?: TransformOptions
}
```

### TransformInstance

Instance created by transform.create().

```typescript
interface TransformInstance {
  // Process method
  process<T>(data: T, options?: TransformOptions): Promise<TransformResult<T>>

  // Batch method
  batch<T>(items: T[], options?: BatchOptions): Promise<TransformResult<T>[]>

  // Close instance
  close(): Promise<void>

  // Get metrics
  getMetrics(): Metrics
}
```

### TransformError

Custom error class.

```typescript
class TransformError extends Error {
  code: string
  details?: any

  constructor(message: string, code: string, details?: any)
}
```

## Constants

### DEFAULT_OPTIONS

Default options used when none provided.

```typescript
const DEFAULT_OPTIONS: TransformOptions = {
  format: 'json',
  validate: true,
  timeout: 30000,
  retries: 3,
}
```

## Events

### Instance Events

Instances emit events for monitoring:

```typescript
instance.on('start', (data) => {
  console.log('Processing started:', data)
})

instance.on('complete', (result) => {
  console.log('Processing completed:', result)
})

instance.on('error', (error) => {
  console.error('Processing error:', error)
})
```
