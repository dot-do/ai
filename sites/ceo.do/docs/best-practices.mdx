# Best Practices

Follow these best practices to build effective CEO Strategic Agent that deliver consistent, high-quality results.

## Agent Design Principles

### 1. Single Responsibility

Each agent should have a clear, focused purpose:

```typescript
// Good: Focused agent
const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  capabilities: [',  // Limited, focused capabilities
  model: 'claude-sonnet-4.5'
})

// Avoid: Over-generalized agent
const overloadedAgent = await agent.create({
  $role: $.GenericAgent,
  capabilities: ['everything'],  // Too broad
  model: 'claude-sonnet-4.5'
})
```

### 2. Rich Context

Provide comprehensive context:

```typescript
// Good: Rich context
const result = await ceoAgent.execute({
  task: 'Specific task',
  context: {
    background: 'Relevant background',
    constraints: ['Constraint 1', 'Constraint 2'],
    goals: 'Specific goals',
    data: {
      /* relevant data */
    },
  },
})

// Avoid: Minimal context
const result = await ceoAgent.execute({
  task: 'Do something', // Too vague
})
```

### 3. Iterative Improvement

Use feedback loops:

```typescript
const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  learning: {
    enabled: true,
    feedback_loop: true,
  },
})

// Provide feedback
await ceoAgent.feedback({
  task_id: result.id,
  outcome: 'success',
  metrics: { quality: 0.95 },
  notes: 'Excellent results',
})
```

## Implementation Patterns

### Event-Driven Architecture

React to business events:

```typescript
import $, { agent, on, send } from 'sdk.do'

const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  model: 'claude-sonnet-4.5',
})

// React to events
await on($.Event.triggered, async (event) => {
  const result = await ceoAgent.execute({
    task: 'Handle event',
    context: event,
  })

  await send($.Result.completed, result)
})
```

### Scheduled Workflows

Automate recurring tasks:

```typescript
import $, { agent, every } from 'sdk.do'

const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  model: 'claude-sonnet-4.5',
})

// Daily automation
await every($.Daily, async () => {
  const report = await ceoAgent.generateReport({
    period: 'yesterday',
    format: 'summary',
  })

  await send($.Report.daily, report)
})

// Weekly planning
await every($.Weekly, async () => {
  const plan = await ceoAgent.planWeek({
    goals: await db.get($.Goals, 'current'),
    resources: await db.get($.Resources, 'available'),
  })

  await db.create($.WeeklyPlan, plan)
})
```

## Error Handling

### Graceful Degradation

Handle errors gracefully:

```typescript
try {
  const result = await ceoAgent.execute(task)
  return result
} catch (error) {
  if (error.code === 'RATE_LIMIT') {
    // Wait and retry
    await sleep(error.retry_after)
    return await ceoAgent.execute(task)
  } else if (error.code === 'TIMEOUT') {
    // Return partial result
    return error.partial_result
  } else {
    // Log and escalate
    await log.error(error)
    throw error
  }
}
```

### Retry Logic

Implement exponential backoff:

```typescript
const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  retry_config: {
    max_retries: 3,
    backoff: 'exponential',
    initial_delay: 1000,
    max_delay: 10000,
  },
})
```

## Security Best Practices

### API Key Management

Secure API key storage:

```typescript
// Use environment variables
const ceoAgent = await agent.create({
  $role: $.ChiefExecutiveOfficer,
  api_keys: {
    anthropic: process.env.ANTHROPIC_API_KEY,
    openai: process.env.OPENAI_API_KEY,
  },
})

// Never hardcode keys
// Bad: api_key: 'sk-...'  // Don't do this!
```

### Input Validation

Validate all inputs:

```typescript
// Input validation
function validateTask(task: any) {
  if (!task.task) throw new Error('Task is required')
  if (!task.context) throw new Error('Context is required')
  if (task.task.length > 10000) throw new Error('Task too long')
  return true
}

const task = validateTask(userInput)
const result = await ceoAgent.execute(task)
```

## Common Pitfalls

### 1. Insufficient Context

**Problem**: Agent produces poor results

**Solution**: Provide comprehensive context

### 2. Over-Reliance on Single Model

**Problem**: Not optimal for all tasks

**Solution**: Use different models for different tasks

### 3. No Error Handling

**Problem**: Application crashes on errors

**Solution**: Implement comprehensive error handling

### 4. Ignoring Rate Limits

**Problem**: API errors due to rate limiting

**Solution**: Implement rate limiting and retry logic
