# soc.org.ai

**An MDXLD-based AI-native superset of [O\*NET](https://www.onetcenter.org/) and Standard Occupational Classification**

soc.org.ai provides a modern, AI-native approach to occupational data by extending the U.S. Department of Labor's O\*NET (Occupational Information Network) with MDXLD linked data features. It offers tree-shakeable, type-safe access to comprehensive occupational information including skills, abilities, knowledge, work activities, and work context.

## Key Features

### ðŸ”— Linked Data with `$` Prefix

Following the MDXLD convention, soc.org.ai uses the `$` prefix for linked data properties:

```typescript
// Traditional occupational data
{
  "code": "15-1252.00",
  "title": "Software Developers",
  "description": "Research, design, and develop computer applications..."
}

// soc.org.ai (MDXLD)
{
  "$type": "Occupation",
  "$id": "https://soc.org.ai/occupations/15-1252.00",
  "onetCode": "15-1252.00",
  "title": "Software Developers",
  "description": "Research, design, and develop computer applications...",
  "skills": [
    { "$ref": "https://soc.org.ai/skills/2.B.4.a" }
  ]
}
```

## Core Concepts

### Occupations

Each occupation includes:

```typescript
interface Occupation {
  $type: 'Occupation'
  $id: string
  onetCode: string // e.g., "15-1252.00"
  title: string
  description: string
  alternativeTitles?: string[]
  tasks?: Task[]
  skills?: Reference<Skill>[]
  abilities?: Reference<Ability>[]
  knowledge?: Reference<Knowledge>[]
}
```

### Skills

Developed capacities that facilitate learning or performance:

```typescript
interface Skill {
  $type: 'Skill'
  $id: string
  elementId: string
  name: string
  description: string
  category: 'Basic Skills' | 'Cross-Functional Skills'
}
```

### Abilities

Enduring attributes of the individual that influence performance:

```typescript
interface Ability {
  $type: 'Ability'
  $id: string
  elementId: string
  name: string
  description: string
  category: 'Cognitive' | 'Physical' | 'Psychomotor' | 'Sensory'
}
```

### Knowledge

Organized sets of principles and facts applying to general domains:

```typescript
interface Knowledge {
  $type: 'Knowledge'
  $id: string
  elementId: string
  name: string
  description: string
}
```

## Usage

### Basic Occupation Lookup

```typescript
import { getOccupation, getOccupationsByTitle } from 'soc.org.ai'

// Get a specific occupation by O*NET code
const softwareDev = getOccupation('15-1252.00')
console.log(softwareDev?.title) // Software Developers

// Search by title
const nurses = getOccupationsByTitle('nurse')
```

### Tree-Shakeable Imports

Import only what you need to minimize bundle size:

```typescript
// Only imports occupation data and functions
import { getOccupation } from 'soc.org.ai/occupations'

// Only imports skill data
import { getSkill, getAllSkills } from 'soc.org.ai/skills'

// Only imports ability data
import { getAbility, getAllAbilities } from 'soc.org.ai/abilities'

// Only imports knowledge data
import { getKnowledge, getAllKnowledge } from 'soc.org.ai/knowledge'

// Import only types (compile-time, zero runtime cost)
import type { Occupation, Skill, Ability } from 'soc.org.ai/types'
```

### MDXLD Integration

Embed occupational data in MDX documents:

```mdx
---
$type: JobPosting
$id: https://example.com/jobs/senior-developer
title: Senior Software Developer
occupation:
  $type: Occupation
  $ref: https://soc.org.ai/occupations/15-1252.00
requiredSkills:
  - $type: Skill
    $ref: https://soc.org.ai/skills/2.B.5.a
    name: Programming
  - $type: Skill
    $ref: https://soc.org.ai/skills/2.A.2.a
    name: Critical Thinking
salaryRange:
  minValue: 120000
  maxValue: 180000
  currency: USD
---

# Senior Software Developer

We're seeking an experienced software developer...
```

## Use Cases

### Career Planning & Job Matching

Help users understand career paths and match skills to occupations:

```typescript
import { getOccupation, getAllOccupations } from 'soc.org.ai'
import { getSkill } from 'soc.org.ai/skills'

// Find occupations requiring specific skills
const programmingSkill = getSkill('2.B.5.a')
const occupations = getAllOccupations().filter((occ) => occ.skills?.some((s) => s.elementId === programmingSkill?.elementId))
```

### Job Posting Enhancement

Create rich, semantically-enhanced job postings:

```typescript
import type { Occupation } from 'soc.org.ai/types'
import { getOccupation } from 'soc.org.ai'

const occupation = getOccupation('15-1252.00')

const jobPosting = {
  $type: 'JobPosting',
  $id: 'https://example.com/jobs/123',
  title: 'Software Developer',
  occupation: {
    $type: 'Occupation',
    $ref: `https://soc.org.ai/occupations/${occupation.onetCode}`,
    name: occupation.title,
  },
  skills: occupation.skills?.map((skill) => ({
    $type: 'DefinedTerm',
    name: skill.name,
  })),
}
```

### Workforce Analytics

Analyze workforce trends and skill gaps:

```typescript
import { getAllOccupations } from 'soc.org.ai'
import { getAllSkills } from 'soc.org.ai/skills'

// Identify most in-demand skills
const skillDemand = new Map()
getAllOccupations().forEach((occ) => {
  occ.skills?.forEach((skill) => {
    skillDemand.set(skill.elementId, (skillDemand.get(skill.elementId) || 0) + 1)
  })
})
```

### AI Agent Integration

Enable AI agents to understand and work with occupational data:

```typescript
import { getOccupation, getOccupationsByTitle } from 'soc.org.ai'

// AI agent processing natural language query
async function findRelatedOccupations(query: string) {
  // Search for occupations matching the query
  const results = getOccupationsByTitle(query)

  // Return structured data for AI processing
  return results.map((occ) => ({
    id: occ.$id,
    title: occ.title,
    description: occ.description,
    skills: occ.skills?.map((s) => s.name),
  }))
}
```

## API Reference

### Occupation Lookup

```typescript
import { getOccupation, getAllOccupations, getOccupationsByTitle, getOccupationsByJobZone, getOccupationsByCodePrefix } from 'soc.org.ai'

// Get specific occupation by O*NET code
const occupation = getOccupation('15-1252.00')

// Get all occupations
const allOccupations = getAllOccupations()

// Search by title (case-insensitive, partial match)
const occupations = getOccupationsByTitle('software')

// Filter by job zone (1-5, indicating education/experience level)
const entryLevel = getOccupationsByJobZone(2)

// Get occupations by SOC code prefix
const computerOccupations = getOccupationsByCodePrefix('15-')
```

### Skills Lookup

```typescript
import { getSkill, getAllSkills } from 'soc.org.ai/skills'

// Get specific skill by element ID
const skill = getSkill('2.B.5.a')

// Get all skills
const allSkills = getAllSkills()
```

### Abilities Lookup

```typescript
import { getAbility, getAllAbilities } from 'soc.org.ai/abilities'

// Get specific ability by element ID
const ability = getAbility('1.A.1.a')

// Get all abilities
const allAbilities = getAllAbilities()
```

### Knowledge Lookup

import { getKnowledge, getAllKnowledge } from 'soc.org.ai/knowledge'

## Why AI-Native Occupational Data?

Traditional occupational classification systems like O\*NET provide invaluable data but were designed for human consumption through reports and web interfaces. soc.org.ai transforms this data for the AI era:

1. **Machine-Readable**: Structured JSON with linked data semantics
2. **Type-Safe**: TypeScript definitions prevent runtime errors
3. **Efficient**: Tree-shakeable exports minimize bundle size
4. **Semantic**: Linked data enables AI systems to understand relationships
5. **Modern DX**: Simple npm install and import, no file parsing needed
