# Getting Started with cli.do

Get up and running with cli.do in under 5 minutes.

## Installation

First, install cli.do globally:

```bash
pnpm add -g cli.do
```

Or use npx without installing:

```bash
npx cli.do --help
```

## Verify Installation

Check that cli.do is installed correctly:

```bash
do --version
```

You should see output like:

```
cli.do v1.0.0
```

## Your First Commands

### 1. List Database Entities

List all businesses in your database:

```bash
do db list Business
```

Output:

```json
[
  {
    "$id": "acme-corp",
    "$type": "Business",
    "name": "Acme Corp",
    "industry": "Technology"
  },
  {
    "$id": "globex",
    "$type": "Business",
    "name": "Globex Corporation",
    "industry": "Manufacturing"
  }
]
```

### 2. Get a Specific Entity

Get details for a specific business:

```bash
do db get Business acme-corp
```

Output:

```json
{
  "$id": "acme-corp",
  "$type": "Business",
  "name": "Acme Corp",
  "industry": "Technology",
  "foundingDate": "2020-01-01",
  "numberOfEmployees": 150
}
```

### 3. Generate Content with AI

Generate text using AI:

```bash
do ai generate "Write a haiku about coding"
```

Output:

```
Code flows like water
Logic branches, functions bloom
Beauty in syntax
```

### 4. Create a New Entity

Create a new person in the database:

```bash
do db create Person '{
  "$id": "jane-smith",
  "$type": "Person",
  "name": "Jane Smith",
  "jobTitle": "Software Engineer"
}'
```

Output:

```json
{
  "$id": "jane-smith",
  "$type": "Person",
  "name": "Jane Smith",
  "jobTitle": "Software Engineer",
  "dateCreated": "2025-10-10T12:00:00Z"
}
```

### 5. Send an Event

Publish a semantic event:

```bash
do send Order.created '{
  "$type": "Order",
  "$id": "order-123",
  "customer": "jane-smith",
  "total": 99.99
}'
```

Output:

```json
{
  "success": true,
  "eventId": "evt_abc123",
  "timestamp": "2025-10-10T12:00:00Z"
}
```

## Configuration

Initialize your configuration file:

```bash
do config init
```

This creates a configuration file at `~/.do/config.yaml`:

```yaml
api:
  endpoint: https://api.do
  key: your-api-key

output:
  format: json
  color: true

log:
  level: info
```

Set individual configuration values:

```bash
# Set API endpoint
do config set api.endpoint https://api.do

# Set API key
do config set api.key your-api-key

# Set default output format
do config set output.format table
```

## Environment Variables

Alternatively, use environment variables:

```bash
# Set API credentials
export DO_API_ENDPOINT=https://api.do
export DO_API_KEY=your-api-key

# Set output preferences
export DO_OUTPUT_FORMAT=json
export DO_LOG_LEVEL=info
```


## Output Formats

cli.do supports multiple output formats:

### JSON (default)

```bash
do db list Business --format json
```

```json
[
  {
    "$id": "acme-corp",
    "$type": "Business",
    "name": "Acme Corp"
  }
]
```

### YAML

```bash
do db list Business --format yaml
```

```yaml
- $id: acme-corp
  $type: Business
  name: Acme Corp
```

### Table

```bash
do db list Business --format table
```

```
┌────────────┬──────────┬────────────┬────────────┐
│ $id        │ $type    │ name       │ industry   │
├────────────┼──────────┼────────────┼────────────┤
│ acme-corp  │ Business │ Acme Corp  │ Technology │
│ globex     │ Business │ Globex     │ Mfg        │
└────────────┴──────────┴────────────┴────────────┘
```

### Pretty (colorized)

```bash
do db list Business --format pretty
```

```
Business: acme-corp
  name: Acme Corp
  industry: Technology
  employees: 150

Business: globex
  name: Globex Corporation
  industry: Manufacturing
  employees: 500
```

## Common Workflows

### Data Exploration

```bash
# List all types
do db types

# Count entities
do db count Business

# Search entities
do db list Business --where '{"industry": "Technology"}'

# Query relationships
do db related Business:acme owns Brand
```

### Content Generation

```bash
# Generate text
do ai generate "Explain quantum computing"

# Generate with specific model
do ai generate "Write a story" --model claude-sonnet-4.5

# Generate structured data
do ai generate "Create a product" --schema Product

# Batch generation
do ai batch --file prompts.json
```

### Event-Driven Workflows

```bash
# Send events
do send Order.created '{"id": "123"}'

# Listen for events
do on Order.created --handler ./process-order.js

# Watch events in real-time
do watch Order.created
```

### Scheduled Tasks

```bash
# Schedule daily task
do every Daily "backup database"

# Schedule hourly task
do every Hourly "collect metrics"

# List scheduled tasks
do every list

# Remove task
do every remove backup-database
```

## Troubleshooting

### Command not found

If you get `command not found: do`, ensure cli.do is installed globally:

```bash
pnpm add -g cli.do
```

Or use the full command:

```bash
npx cli.do db list Business
```

### Authentication errors

If you get authentication errors, set your API key:

```bash
do config set api.key your-api-key
```

Or use an environment variable:

```bash
export DO_API_KEY=your-api-key
```

### Connection errors

If you get connection errors, check your API endpoint:

```bash
do config get api.endpoint
```

Set the correct endpoint:

```bash
do config set api.endpoint https://api.do
```

### JSON parsing errors

If you get JSON parsing errors, ensure your JSON is valid:

```bash
# Bad (single quotes won't work for JSON values)
do db create Person '{"name": 'John'}'

# Good (use double quotes)
do db create Person '{"name": "John"}'

# Alternative (use file input)
echo '{"name": "John"}' > person.json
do db create Person --file person.json
```
