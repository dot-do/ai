# Installation Guide

Complete guide to installing and configuring cli.do on your system.

## System Requirements

- **Node.js**: 18.0.0 or higher
- **Package Manager**: pnpm (recommended), npm, or yarn
- **Operating System**: macOS, Linux, or Windows (WSL recommended)
- **Terminal**: Any modern terminal with ANSI color support

## Installation Methods

### Global Installation (Recommended)

Install cli.do globally to use the `do` command anywhere:

#### Using pnpm (Recommended)

```bash
pnpm add -g cli.do
```

#### Using npm

```bash
npm install -g cli.do
```

#### Using yarn

```bash
yarn global add cli.do
```

### Project-Local Installation

Install cli.do as a dev dependency in your project:

```bash
pnpm add -D cli.do
```

Then use via npx:

```bash
npx do db list Business
```

Or add scripts to your `package.json`:

```json
{
  "scripts": {
    "db:list": "do db list",
    "ai:generate": "do ai generate"
  }
}
```

### Using npx (No Installation)

Use cli.do without installing:

```bash
npx cli.do db list Business
npx cli.do ai generate "Write a haiku"
```

Note: This downloads cli.do on each run, so it's slower than global installation.

## Verify Installation

Check that cli.do is installed correctly:

```bash
do --version
```

Expected output:

```
cli.do v1.0.0
sdk.do v1.0.0
Node.js v20.10.0
```

Test basic functionality:

```bash
do --help
```

You should see the help menu with available commands.

## Configuration

### Initialize Configuration

Create your configuration file:

```bash
do config init
```

This creates `~/.do/config.yaml`:

```yaml
api:
  endpoint: https://api.do
  key: ''

output:
  format: json
  color: true
  pretty: false

log:
  level: info
  file: ~/.do/logs/cli.log

editor:
  command: code
  args: []
```

### Set API Credentials

#### Option 1: Using CLI

```bash
do config set api.endpoint https://api.do
do config set api.key your-api-key-here
```

#### Option 2: Environment Variables

Add to your `~/.bashrc`, `~/.zshrc`, or `~/.profile`:

```bash
export DO_API_ENDPOINT=https://api.do
export DO_API_KEY=your-api-key-here
```

Reload your shell:

```bash
source ~/.bashrc  # or ~/.zshrc
```

#### Option 3: .env File

Create a `.env` file in your project:

```env
DO_API_ENDPOINT=https://api.do
DO_API_KEY=your-api-key-here
DO_OUTPUT_FORMAT=json
DO_LOG_LEVEL=info
```

cli.do will automatically load this file.

#### Option 4: Manual Configuration

Edit the config file directly:

```bash
do config edit
```

Or manually edit `~/.do/config.yaml`:

```yaml
api:
  endpoint: https://api.do
  key: your-api-key-here
```

### Get API Key

To get your API key:

1. Visit [https://platform.do/api-keys](https://platform.do/api-keys)
2. Sign in or create an account
3. Click "Create API Key"
4. Copy the key and save it securely

### View Configuration

View current configuration:

```bash
do config get
```

View specific value:

```bash
do config get api.endpoint
do config get output.format
```

### Multiple Environments

cli.do supports multiple environment configurations:

```bash
# Create environment-specific configs
do config init --env production
do config init --env staging
do config init --env development

# Use specific environment
do --env production db list Business
do --env staging ai generate "test"

# Set default environment
do config set env.default production
```

Configuration files:

- `~/.do/config.yaml` - Default configuration
- `~/.do/config.production.yaml` - Production environment
- `~/.do/config.staging.yaml` - Staging environment
- `~/.do/config.development.yaml` - Development environment

## Shell Completion

Enable shell autocompletion for faster command entry.

### Bash

```bash
# Add to ~/.bashrc
eval "$(do completion bash)"

# Or generate completion file
do completion bash > /etc/bash_completion.d/do
```

### Zsh

```bash
# Add to ~/.zshrc
eval "$(do completion zsh)"

# Or generate completion file
do completion zsh > /usr/local/share/zsh/site-functions/_do
```

### Fish

```bash
# Add to ~/.config/fish/config.fish
do completion fish | source

# Or generate completion file
do completion fish > ~/.config/fish/completions/do.fish
```

### PowerShell

```powershell
# Add to $PROFILE
Invoke-Expression (do completion powershell | Out-String)
```

## Aliases

Create convenient aliases for common commands:

### Bash/Zsh

Add to `~/.bashrc` or `~/.zshrc`:

```bash

alias dbg="do db get"
alias dbc="do db db create"

# AI shortcuts
alias gen="do ai generate"
alias embed="do ai embed"

# Event shortcuts
alias send="do send"
alias watch="do watch"

# Utility shortcuts
alias dorepl="do repl"
alias doconf="do config"
```

### Fish

Add to `~/.config/fish/config.fish`:

```fish

alias dbl "do db list"
alias dbg "do db get"
alias dbc "do db create"

# AI shortcuts
alias gen "do ai generate"
alias embed "do ai embed"
```

## PATH Configuration

If `do` command is not found after global installation:

### macOS/Linux

Add to `~/.bashrc`, `~/.zshrc`, or `~/.profile`:

```bash
# pnpm global bin
export PATH="$HOME/.local/share/pnpm:$PATH"

# npm global bin
export PATH="$(npm config get prefix)/bin:$PATH"
```

### Windows

1. Open System Properties > Environment Variables
2. Edit PATH variable
3. Add pnpm or npm global bin path:
   - pnpm: `%APPDATA%\npm`
   - npm: `%APPDATA%\npm`

## Docker Installation

Run cli.do in Docker without installing:

```bash
# Pull image
docker pull dotdo/cli.do

# Run command
docker run --rm -it dotdo/cli.do db list Business

# Interactive mode
docker run --rm -it dotdo/cli.do repl

# With mounted config
docker run --rm -it -v ~/.do:/root/.do dotdo/cli.do db list Business
```

Create an alias for convenience:

```bash
alias do="docker run --rm -it -v ~/.do:/root/.do dotdo/cli.do"
```

## Troubleshooting

### Command Not Found

**Problem**: `command not found: do`

**Solutions**:

1. Verify installation:

   ```bash
   pnpm list -g | grep cli.do
   ```

2. Check PATH:

   ```bash
   echo $PATH
   ```

3. Reinstall globally:
   ```bash
   pnpm remove -g cli.do
   pnpm add -g cli.do
   ```

### Permission Errors

**Problem**: `EACCES: permission denied`

**Solutions**:

1. Use pnpm (doesn't require sudo):

   ```bash
   pnpm add -g cli.do
   ```

2. Fix npm permissions:

   ```bash
   mkdir ~/.npm-global
   npm config set prefix '~/.npm-global'
   export PATH=~/.npm-global/bin:$PATH
   ```

3. Use project-local installation:
   ```bash
   pnpm add -D cli.do
   npx do --help
   ```

### Outdated Node.js

**Problem**: `Error: Node.js version 18.0.0 or higher required`

**Solutions**:

1. Update Node.js:

   ```bash
   # Using nvm
   nvm install 20
   nvm use 20

   # Using Homebrew (macOS)
   brew upgrade node
   ```

2. Use Docker:
   ```bash
   docker run --rm -it dotdo/cli.do
   ```

### Authentication Errors

**Problem**: `401 Unauthorized`

**Solutions**:

1. Set API key:

   ```bash
   do config set api.key your-api-key
   ```

2. Check API key:

   ```bash
   do config get api.key
   ```

3. Regenerate API key at [https://platform.do/api-keys](https://platform.do/api-keys)

### Network Errors

**Problem**: `ECONNREFUSED` or `ETIMEDOUT`

**Solutions**:

1. Check API endpoint:

   ```bash
   do config get api.endpoint
   ```

2. Test connectivity:

   ```bash
   curl https://api.do/health
   ```

3. Check firewall/proxy settings
