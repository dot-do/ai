# API Overview

sdk.do provides a comprehensive API for building Business-as-Code applications.

## Modules

### $ (Semantic Proxy)

Access 800+ Schema.org types for semantic operations.

```typescript
import $ from 'sdk.do'

$.Business
$.Person
$.Product
$.Order
```

[Learn more →](./functions#dollar-proxy)

### db (Database)

CRUD operations and semantic relationships.

```typescript
import { db } from 'sdk.do'

await db.create($.Business, data)
await db.list($.Person, { where: { ... } })
await db.relate(person, $.worksFor, organization)
```

[Learn more →](./functions#db)

### ai (AI Services)

AI-powered generation, embeddings, and batch processing.

```typescript
import { ai } from 'sdk.do'

await ai.generate('Create content', { schema: $.Product })
await ai.embed('Search query')
await ai.batch([...])
```

[Learn more →](./functions#ai)

### api (External APIs)

HTTP requests to external services.

```typescript
import { api } from 'sdk.do'

await api('https://api.example.com/data', {
  method: 'POST',
  body: data,
})
```

[Learn more →](./functions#api)

### on (Event Listeners)

Listen for semantic events.

```typescript
import { on } from 'sdk.do'

on($.Order.created, async (order) => {
  // Handle event
})
```

[Learn more →](./functions#on)

### send (Event Publishing)

Publish semantic events.

```typescript
import { send } from 'sdk.do'

send($.Order.created, {
  $type: 'Order',
  $id: 'order-123',
})
```

[Learn more →](./functions#send)

### every (Scheduled Workflows)

Execute functions on a schedule.

```typescript
import { every } from 'sdk.do'

every($.Daily, async () => {
  // Run daily
})
```

[Learn more →](./functions#every)

### user (User Context)

Access current user and permissions.

```typescript
import { user } from 'sdk.do'

const currentUser = await user.current()
const hasPermission = await user.can('admin')
```

[Learn more →](./functions#user)

## Quick Reference

### Database Operations

| Function                                | Description         |
| --------------------------------------- | ------------------- |
| `db.list(type, options)`                | List entities       |
| `db.get(type, id)`                      | Get entity by ID    |
| `db.create(type, data)`                 | Create entity       |
| `db.update(entity, data)`               | Update entity       |
| `db.delete(entity)`                     | Delete entity       |
| `db.relate(subject, predicate, object)` | Create relationship |
| `db.related(subject, predicate, type)`  | Query relationships |

### AI Operations

| Function                       | Description       |
| ------------------------------ | ----------------- |
| `ai.generate(prompt, options)` | Generate content  |
| `ai.embed(text)`               | Create embeddings |
| `ai.batch(requests)`           | Batch processing  |

### Event Operations

| Function             | Description       |
| -------------------- | ----------------- |
| `on(event, handler)` | Listen for events |
| `send(event, data)`  | Publish events    |

### Workflow Operations

| Function                   | Description       |
| -------------------------- | ----------------- |
| `every(schedule, handler)` | Schedule workflow |

## Environment Configuration

Configure sdk.do with environment variables:

```bash
SDK_DATABASE_URL=postgres://...
SDK_OPENAI_API_KEY=sk-...
SDK_AI_MODEL=gpt-5
```

## Error Handling

All async functions can throw errors:

```typescript
try {
  await db.create($.Business, data)
} catch (error) {
  console.error('Failed:', error)
}
```

## Rate Limiting

AI operations are rate-limited. Use batch processing:

```typescript
// Good - batch processing
await ai.batch(requests)

// Avoid - sequential requests
for (const req of requests) {
  await ai.generate(req.prompt)
}
```

## Best Practices

1. **Use semantic patterns** - `$.Subject.predicate.Object`
2. **Batch AI operations** - 50% cost savings
3. **Handle errors** - Always use try-catch
4. **Type your data** - Use TypeScript types
5. **Leverage relationships** - Use `db.relate` and `db.related`
