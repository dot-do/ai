# gs1.org.ai

**An MDXLD-based AI-native extension of [GS1](https://www.gs1.org) standards**

gs1.org.ai extends GS1 standards for product identification, barcodes, and supply chain data with AI-native features, using the [MDXLD](/mdxld) format for linked data in MDX. It provides a modern, type-safe, and AI-friendly way to work with GS1 identifiers and product data.

## Key Features

> **Note**: The `gs1.org.ai` package is currently in development. This documentation describes the planned API and features. Some functionality may not yet be implemented.

gs1.org.ai is designed from the ground up for AI systems:

- **TypeScript-first**: Full type safety with generated TypeScript definitions
- **MDX Integration**: Native support for embedded GS1 data in MDX documents
- **Validation**: Built-in validation for GTINs and other GS1 identifiers _(planned)_

### ðŸ“¦ GS1 Standards Support

Support for key GS1 standards:

- **GTIN** (Global Trade Item Number) - Product identification
- **GLN** (Global Location Number) - Location identification
- **SSCC** (Serial Shipping Container Code) - Logistics units
- **GRAI** (Global Returnable Asset Identifier) - Returnable assets
- **GIAI** (Global Individual Asset Identifier) - Individual assets
- **GSRN** (Global Service Relation Number) - Service relationships

## MDXLD Integration

MDXLD enables `$id` and `$type` in MDX frontmatter for GS1 linked data:

```mdx
---
$id: https://id.gs1.org/01/00614141000012
$type: Product
gtin: '00614141000012'
name: Organic Coffee Beans
brand: Example Brand
netContent:
  $type: QuantitativeValue
  value: 500
  unitCode: GRM
---

# Organic Coffee Beans

Product information and details...
```

The frontmatter is automatically parsed and validated as GS1 linked data, making it easy for AI systems to understand and process product information.

## Core Packages

### `gs1.org.ai`

Tree-shakeable GS1 types and data with full TypeScript support.

```bash
pnpm add gs1.org.ai
```

Features:

- Full GS1 identifier support with `$` prefix
- Tree-shakeable exports
- TypeScript type definitions
- GTIN validation utilities
- Product and location lookup

### `mdxld`

Enable `$id` and `$type` in MDX frontmatter for linked data.

```bash
pnpm add mdxld
```

Features:

- Parse and validate MDXLD frontmatter
- Convert between nested and flattened formats
- JSON-LD conversion
- TypeScript support

## Use Cases

### Product Catalogs

Embed GS1 product data directly in your MDX content:

```mdx
---
$type: Product
$id: https://id.gs1.org/01/00614141000012
gtin: '00614141000012'
name: Premium Coffee Blend
brand: Coffee Co
manufacturer:
  $type: Organization
  name: Coffee Manufacturing Inc
  gln: '0614141000005'
netContent:
  $type: QuantitativeValue
  value: 1
  unitCode: KGM
---

# Premium Coffee Blend

Rich, aromatic coffee sourced from sustainable farms...
```

### Supply Chain Tracking

Track products through the supply chain with GS1 identifiers:

```typescript
import type { Product, Location } from 'gs1.org.ai/types'

const shipment = {
  $type: 'ParcelDelivery',
  $id: 'https://id.gs1.org/00/006141410000000001',
  trackingNumber: '006141410000000001',
  itemShipped: {
    $type: 'Product',
    gtin: '00614141000012',
  },
  originAddress: {
    $type: 'Location',
    gln: '0614141000005',
  },
}
```

### E-commerce Integration

Integrate GS1 standards with e-commerce platforms:

```typescript
import { validateGTIN, parseGTIN } from 'gs1.org.ai'

const gtin = '00614141000012'
const isValid = validateGTIN(gtin)
const parsed = parseGTIN(gtin)

console.log(parsed)
// {
//   valid: true,
//   type: 'GTIN-14',
//   indicator: '0',
//   companyPrefix: '061414',
//   itemReference: '100001',
//   checkDigit: '2'
// }
```

## API Reference

> **Note**: The following APIs are planned and not yet implemented. This section describes the intended functionality of the `gs1.org.ai` package.

### GTIN Validation

```typescript
import { validateGTIN, parseGTIN, calculateCheckDigit } from 'gs1.org.ai'

// Validate GTIN
const isValid = validateGTIN('00614141000012')

// Parse GTIN structure
const parsed = parseGTIN('00614141000012')

// Calculate check digit
const checkDigit = calculateCheckDigit('0061414100001')
```

### Identifier Lookup

```typescript
import { getProduct, getLocation, validateGLN } from 'gs1.org.ai'

// Get product by GTIN
const product = await getProduct('00614141000012')

// Get location by GLN
const location = await getLocation('0614141000005')

// Validate GLN
const isValidGLN = validateGLN('0614141000005')
```

### MDXLD Parsing

```typescript
import { parseMDXLD, validateLinkedData, toJSONLD } from 'mdxld'

// Parse MDX with GS1 linked data
const parsed = parseMDXLD(mdxContent)

// Validate
const validation = validateLinkedData(parsed.data, {
  validateId: true,
  requireType: true,
})

// Convert to JSON-LD
const jsonld = toJSONLD(parsed.data)
```
