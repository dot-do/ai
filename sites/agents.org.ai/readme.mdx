# agents.org.ai

**An open-source ontology of AI agents and their capabilities in Business-as-Code**

agents.org.ai provides a standardized vocabulary for defining AI agents, their capabilities, and behaviors in Business-as-Code systems. It enables consistent agent configuration, discovery, and orchestration through a semantic ontology of agent types, skills, and interactions.

## Core Concepts

### Agents

Agents represent autonomous AI entities with specific capabilities and goals:

- **Task Agents**: Specialized for specific tasks (e.g., CodeReviewAgent, DataAnalysisAgent)
- **Conversational Agents**: Interactive assistants for user communication
- **Orchestrator Agents**: Meta-agents that coordinate other agents
- **Monitoring Agents**: Agents that observe and report on system state

### Agent Types

```typescript
{
  "$type": "AgentType",
  "$id": "https://agents.org.ai/types/Assistant",
  "name": "Assistant",
  "description": "Interactive agent that assists users with tasks",
  "characteristics": [
    "conversational",
    "task-oriented",
    "context-aware"
  ],
  "examples": [
    { "$ref": "https://agents.org.ai/ClaudeAssistant" },
    { "$ref": "https://agents.org.ai/CursorAgent" }
  ]
}
```

### Capabilities

Capabilities define what an agent can do:

- **Analysis Capabilities**: Code review, data analysis, pattern recognition
- **Generation Capabilities**: Content creation, code generation, synthesis
- **Interaction Capabilities**: User communication, API integration, tool usage
- **Orchestration Capabilities**: Workflow management, agent coordination, task delegation

### Tools

Tools are the concrete functions agents can invoke:

```typescript
{
  "$type": "AgentTool",
  "$id": "https://agents.org.ai/tools/Grep",
  "name": "Grep",
  "description": "Search for patterns in files",
  "parameters": {
    "pattern": "string",
    "path": "string",
    "options": "object"
  },
  "returns": {
    "$type": "SearchResults",
    "matches": "array"
  }
}
```

## Use Cases

### Agent Discovery and Selection

Enable systems to discover and select appropriate agents for tasks:

```typescript
import { findAgent, getAgentCapabilities } from 'agents.org.ai'

// Find agents with specific capabilities
const codeReviewAgents = await findAgent({
  capabilities: ['CodeAnalysis', 'SecurityAudit'],
})

// Get agent capabilities
const capabilities = await getAgentCapabilities('CodeReviewAgent')
```

### Agent Orchestration

Coordinate multiple agents to accomplish complex tasks:

```mdx
---
$type: AgentWorkflow
$id: https://example.com/workflows/pr-review
name: Pull Request Review
description: Multi-agent PR review workflow
agents:
  - $type: Agent
    role: SecurityReviewer
    capabilities: [SecurityAudit, VulnerabilityDetection]
  - $type: Agent
    role: CodeReviewer
    capabilities: [CodeAnalysis, StyleChecking]
  - $type: Agent
    role: TestReviewer
    capabilities: [TestAnalysis, CoverageCheck]
---

# Pull Request Review Workflow

This workflow orchestrates multiple specialized agents to perform comprehensive PR reviews...
```

### Agent Configuration

Define agent behavior and constraints declaratively:

```typescript
{
  "$type": "AgentConfig",
  "$id": "https://example.com/agents/my-agent",
  "name": "MyCustomAgent",
  "baseAgent": {
    "$ref": "https://agents.org.ai/Assistant"
  },
  "instructions": "You are a helpful assistant specialized in...",
  "constraints": [
    "Always cite sources",
    "Never execute destructive commands",
    "Validate input before processing"
  ],
  "tools": [
    { "$ref": "https://actions.org.ai/Read" },
    { "$ref": "https://actions.org.ai/Write" },
    { "$ref": "https://actions.org.ai/Grep" }
  ],
  "model": "claude-3-5-sonnet-20241022",
  "temperature": 0.7,
  "maxTokens": 4096
}
```

## API Reference

### Agent Lookup

```typescript
import { getAgent, getAllAgents, getAgentsByCapability } from 'agents.org.ai'

// Get specific agent
const agent = getAgent('CodeReviewAgent')

// Get all agents
const allAgents = getAllAgents()

// Get agents by capability
const analysisAgents = getAgentsByCapability('CodeAnalysis')
```

### Capability Lookup

```typescript
import { getCapability, getAllCapabilities, getCapabilitiesByAgent } from 'agents.org.ai'

// Get specific capability
const capability = getCapability('CodeAnalysis')

// Get all capabilities
const allCapabilities = getAllCapabilities()

// Get capabilities for an agent
const agentCapabilities = getCapabilitiesByAgent('CodeReviewAgent')
```

### Agent Instantiation

```typescript
import { createAgent, configureAgent, validateAgent } from 'agents.org.ai'

// Create agent instance
const agent = createAgent('CodeReviewAgent', {
  model: 'claude-3-5-sonnet-20241022',
  tools: ['grep', 'lint', 'test'],
})

// Configure agent
const configured = configureAgent(agent, {
  temperature: 0.7,
  maxTokens: 4096,
  systemPrompt: 'You are an expert code reviewer...',
})

// Validate agent configuration
const validation = validateAgent(configured)
```

## Agent Categories

Agents are organized into semantic categories:

- **Development**: Code review, testing, debugging, refactoring
- **Data**: Analysis, transformation, validation, enrichment
- **Communication**: Customer support, documentation, translation
- **Operations**: Monitoring, incident response, deployment
- **Research**: Information gathering, summarization, synthesis
- **Creative**: Content generation, design, ideation

## Configuration Formats

agents.org.ai supports multiple agent configuration formats:

### Universal Format (agents.md)

```markdown
# MyAgent

AI assistant for code review

## Instructions

Review code for security, performance, and best practices

## Tools

- grep: Search code for patterns
- lint: Run static analysis
- test: Execute test suite

## Rules

- Check for SQL injection vulnerabilities
- Ensure proper error handling
- Follow team style guide
```

### Claude Format (CLAUDE.md)

```markdown
# MyAgent

You are an expert code reviewer...

## Guidelines

1. Review for security issues
2. Check performance
3. Validate best practices
```

### Cursor Format (.cursorrules)

```
You are a code review assistant.

Focus on:
- Security vulnerabilities
- Performance issues
- Best practices
```

### JSON Format

```json
{
  "$type": "Agent",
  "name": "MyAgent",
  "instructions": "Review code for security, performance, and best practices",
  "tools": ["grep", "lint", "test"],
  "rules": ["Check for SQL injection vulnerabilities", "Ensure proper error handling", "Follow team style guide"]
}
```

## Agent Lifecycle

### 1. Definition

Define agent capabilities and behavior:

```typescript
const agentDef = {
  $type: 'Agent',
  name: 'MyAgent',
  capabilities: ['CodeAnalysis'],
  tools: ['grep', 'lint'],
}
```

### 2. Configuration

Configure agent for specific contexts:

```typescript
const config = {
  model: 'claude-3-5-sonnet-20241022',
  temperature: 0.7,
  systemPrompt: 'You are an expert...',
}
```

### 3. Instantiation

Create agent instance:

```typescript
const agent = createAgent(agentDef, config)
```

### 4. Execution

Execute agent tasks:

```typescript
const result = await agent.execute({
  task: 'review-code',
  input: codeContext,
})
```

### 5. Monitoring

Monitor agent performance:

```typescript
const metrics = await agent.getMetrics()
```
