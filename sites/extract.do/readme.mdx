# extract.do

**Extract, parse, and transform data from HTML, JSON, XML, CSV and other sources**

extract.do provides powerful utilities for data extraction in your Business-as-Code applications. Built with semantic patterns, type safety, and integration with the entire `.do` platform ecosystem.

## Quick Start

```bash
pnpm add extract.do
```

```typescript
import $, { extract } from 'extract.do'

// Basic usage
const result = await extract({
  // Your data
  data: 'example',
  // Options
  options: {},
})

console.log('Result:', result)
```

## Key Features

### Semantic Patterns

Use `$.Subject.predicate.Object` patterns for type-safe operations:

```typescript
import $ from 'extract.do'

// Semantic patterns
const result = await extract($.Data, {
  // Configuration
})
```

### Type Safety

Full TypeScript support with comprehensive type definitions:

```typescript
import type { ExtractOptions, ExtractResult } from 'extract.do'

const options: ExtractOptions = {
  // Type-safe configuration
}

const result: ExtractResult = await extract(options)
```

### Integration Ready

Seamlessly integrates with other `.do` platform services:

```typescript
import { extract } from 'extract.do'
import { db, ai } from 'sdk.do'

// Integrate with database
const data = await db.list($.Data)
const result = await extract(data)

// Integrate with AI
const processed = await ai.generate('process', { data: result })
```

## TypeScript Support

extract.do is built with TypeScript and provides full type safety:

```typescript
import type { ExtractOptions, ExtractResult, ExtractError } from 'extract.do'

// Type-safe operations
const options: ExtractOptions = {
  // Configuration
}

try {
  const result: ExtractResult = await extract(options)

} catch (error: ExtractError) {
  console.error('Error:', error.message)
}
```

## Best Practices

### Use Semantic Patterns

```typescript
// Good - semantic patterns
const result = await extract($.Data, options)

// Avoid - string-based
const result = await extract('Data', options)
```

### Handle Errors

```typescript
// Good - error handling
try {
  const result = await extract(data)
  return result
} catch (error) {
  console.error('Failed:', error)
  throw error
}
```

### Use TypeScript

```typescript
// Good - type-safe
import type { Options } from 'extract.do'
const options: Options = {}

// Avoid - untyped
const options = {}
```
