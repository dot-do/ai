# Basic CTO Agent Usage

Simple examples to get started with CTO agents.

## Example 1: Technology Evaluation

```typescript
import $, { agent } from 'sdk.do'

const cto = await agent.create({
  $role: $.ChiefTechnologyOfficer,
  name: 'Tech Evaluator',
  model: 'claude-sonnet-4.5',
})

const evaluation = await cto.execute({
  task: 'Should we adopt TypeScript for our JavaScript codebase?',
  context: {
    current: 'JavaScript ES6, 50K lines of code',
    team: '12 engineers, varying JS expertise',
    pain_points: ['Runtime errors', 'Refactoring difficulties', 'Large codebase maintainability'],
  },
})

console.log(evaluation.recommendation)
console.log(evaluation.implementation_plan)
```

## Example 2: Architecture Review

```typescript
const review = await cto.execute({
  task: 'Review proposed microservices architecture',
  proposal: {
    services: ['Auth', 'Payment', 'Notification', 'Analytics'],
    technology: 'Node.js, PostgreSQL per service, Kafka',
    infrastructure: 'Kubernetes on AWS',
  },
  context: {
    team_size: 20,
    current: 'Rails monolith',
    timeline: '6 months',
  },
})
```

## Example 3: Team Structure

```typescript
const teamPlan = await cto.execute({
  task: 'Design team structure for scaling from 15 to 40 engineers',
  context: {
    current: '3 feature teams, 1 platform team',
    products: ['Web app', 'Mobile app', 'API platform'],
    challenges: ['Cross-team dependencies', 'Knowledge silos'],
  },
})
```
