# Schedules

Complete guide to all schedule types supported by every.do, including traditional cron patterns and semantic schedules.

## Semantic Schedules

Semantic schedules use human-readable patterns from the `$` semantic proxy, making code more intuitive and AI-understandable.

### Time-Based Schedules

#### Minute-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// Every minute
every($.Minutely, async () => {
  await checkHighFrequencyData()
})

// Every 5 minutes
every($.Every5Minutes, async () => {
  await monitorSystemHealth()
})

// Every 15 minutes
every($.Every15Minutes, async () => {
  await syncRealTimeInventory()
})

// Every 30 minutes
every($.Every30Minutes, async () => {
  await refreshCache()
})
```

#### Hour-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// Every hour (on the hour)
every($.Hourly, async () => {
  await syncData()
})

// Every 2 hours
every($.Every2Hours, async () => {
  await processQueue()
})

// Every 6 hours
every($.Every6Hours, async () => {
  await fullDataSync()
})

// Every 12 hours
every($.Every12Hours, async () => {
  await runDiagnostics()
})
```

#### Day-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// Every day at midnight UTC
every($.Daily, async () => {
  await generateDailyReport()
})

// Every business day (Mon-Fri) at 9am UTC
every($.BusinessDay, async () => {
  await sendMorningBriefing()
})

// Every weekend (Sat-Sun) at 9am UTC
every($.Weekend, async () => {
  await runWeekendMaintenance()
})
```

#### Week-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// Every Monday at midnight UTC
every($.Weekly, async () => {
  await generateWeeklyReport()
})

// Every other Monday at midnight UTC
every($.BiWeekly, async () => {
  await processPayroll()
})
```

#### Month-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// First day of month at midnight UTC
every($.Monthly, async () => {
  await processMonthlyBilling()
})

// Last day of month at 11pm UTC

})
```

#### Quarter-Level Schedules

```typescript
import { $, every } from 'sdk.do'

// Last day of quarter at 11pm UTC
```

#### Year-Level Schedules

```typescript

// January 1st at midnight UTC

// December 31st at 11pm UTC

```

## Cron Patterns

Traditional cron syntax provides precise control over scheduling.

### Cron Syntax

```
 ┌───────────── minute (0 - 59)
 │ ┌───────────── hour (0 - 23)
 │ │ ┌───────────── day of month (1 - 31)
 │ │ │ ┌───────────── month (1 - 12)
 │ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
 │ │ │ │ │
 │ │ │ │ │
 * * * * *
```

### Special Characters

- **`*`** - Any value (wildcard)
- **`,`** - Value list separator
- **`-`** - Range of values
- **`/`** - Step values

### Common Cron Patterns

#### Every Minute

```typescript
import { every } from 'sdk.do'

// Every minute
every('* * * * *', async () => {
  await monitorCriticalSystem()
})
```

#### Every N Minutes

```typescript
import { every } from 'sdk.do'

// Every 5 minutes
every('*/5 * * * *', async () => {
  await checkStatus()
})

// Every 10 minutes
every('*/10 * * * *', async () => {
  await syncData()
})

// Every 15 minutes
every('*/15 * * * *', async () => {
  await refreshCache()
})

// Every 30 minutes
every('*/30 * * * *', async () => {
  await processQueue()
})
```

#### Specific Minutes Each Hour

```typescript
import { every } from 'sdk.do'

// At :15 and :45 past every hour
every('15,45 * * * *', async () => {
  await halfHourlyTask()
})

// At :00, :20, :40 past every hour
every('0,20,40 * * * *', async () => {
  await frequentTask()
})
```

#### Every Hour

```typescript
import { every } from 'sdk.do'

// Every hour on the hour
every('0 * * * *', async () => {
  await hourlyTask()
})

// Every 2 hours
every('0 */2 * * *', async () => {
  await biHourlyTask()
})

// Every 3 hours
every('0 */3 * * *', async () => {
  await triHourlyTask()
})

// Every 6 hours
every('0 */6 * * *', async () => {
  await quarterDailyTask()
})
```

#### Specific Times Daily

```typescript
import { every } from 'sdk.do'

// Every day at midnight
every('0 0 * * *', async () => {
  await dailyTask()
})

// Every day at 2:30am
every('30 2 * * *', async () => {
  await backupDatabase()
})

// Every day at 9:00am
every('0 9 * * *', async () => {
  await morningReport()
})

// Every day at 6:00pm
every('0 18 * * *', async () => {
  await eveningReport()
})

// Multiple times per day (9am, 12pm, 3pm, 6pm)
every('0 9,12,15,18 * * *', async () => {
  await businessHoursCheck()
})
```

#### Specific Days of Week

```typescript
import { every } from 'sdk.do'

// Mondays at 9am
every('0 9 * * 1', async () => {
  await mondayMorning()
})

// Fridays at 5pm
every('0 17 * * 5', async () => {
  await fridayAfternoon()
})

// Weekdays (Mon-Fri) at 8am
every('0 8 * * 1-5', async () => {
  await businessDayMorning()
})

// Weekends (Sat-Sun) at 10am
every('0 10 * * 0,6', async () => {
  await weekendMorning()
})

// Every Sunday at midnight
every('0 0 * * 0', async () => {
  await weeklyCleanup()
})
```

#### Specific Days of Month

```typescript
import { every } from 'sdk.do'

// First day of month at midnight
every('0 0 1 * *', async () => {
  await monthlyBilling()
})

// 15th of month at noon
every('0 12 15 * *', async () => {
  await midMonthReport()
})

// Last day of month (approximation - runs on 28th)
every('0 23 28 * *', async () => {
  await monthEndTask()
})

// 1st and 15th of month
every('0 0 1,15 * *', async () => {
  await biMonthlyTask()
})
```

#### Specific Months

```typescript
import { every } from 'sdk.do'

// First day of January at midnight
every('0 0 1 1 *', async () => {
  await newYearProcessing()
})

// First Monday of every month at 9am
every('0 9 1-7 * 1', async () => {
  await firstMondayTask()
})

// Quarterly (Jan, Apr, Jul, Oct) on 1st at midnight
every('0 0 1 1,4,7,10 *', async () => {
  await quarterlyReport()
})
```

| Pattern       | Description         | Example Time              |
| ------------- | ------------------- | ------------------------- |
| `* * * * *`   | Every minute        | Every minute              |
| `*/5 * * * *` | Every 5 minutes     | :00, :05, :10, etc.       |
| `0 * * * *`   | Every hour          | 00:00, 01:00, 02:00, etc. |
| `0 */2 * * *` | Every 2 hours       | 00:00, 02:00, 04:00, etc. |
| `0 0 * * *`   | Daily at midnight   | 00:00 every day           |
| `30 2 * * *`  | Daily at 2:30am     | 02:30 every day           |
| `0 9 * * 1-5` | Weekdays at 9am     | 09:00 Mon-Fri             |
| `0 0 * * 0`   | Sundays at midnight | 00:00 Sunday              |
| `0 0 1 * *`   | First of month      | 00:00 on day 1            |
| `0 0 1 1 *`   | New Year's Day      | 00:00 Jan 1               |

## Timezone Support

By default, all schedules run in UTC. Specify a timezone for local time execution:

### Timezone Configuration

```typescript
import { $, every } from 'sdk.do'

// Run at 9am New York time
every(
  $.Daily,
  async () => {
    await morningReport()
  },
  {
    timezone: 'America/New_York',
  }
)

// Run at 6pm London time
every(
  '0 18 * * *',
  async () => {
    await endOfDayProcessing()
  },
  {
    timezone: 'Europe/London',
  }
)

// Run at 2pm Tokyo time
every(
  '0 14 * * *',
  async () => {
    await afternoonSync()
  },
  {
    timezone: 'Asia/Tokyo',
  }
)
```

| Eastern Time  | `America/New_York`    |
| Central Time  | `America/Chicago`     |

| Pacific Time  | `America/Los_Angeles` |
| **Europe**    |                       |
| London        | `Europe/London`       |
| Paris         | `Europe/Paris`        |
| Berlin        | `Europe/Berlin`       |
| **Asia**      |                       |
| Tokyo         | `Asia/Tokyo`          |

### Environment Variable Timezone

Set default timezone for all schedules:

```bash
# .env
SCHEDULE_TIMEZONE=America/New_York
```

```typescript
import { $, every } from 'sdk.do'

// Uses SCHEDULE_TIMEZONE if set
every($.Daily, async () => {
  await task()
})

// Override environment timezone
every(
  $.Daily,
  async () => {
    await task()
  },
  {
    timezone: 'UTC',
  }
)
```

## Schedule Comparison

### Semantic vs Cron

Choose the right approach for your use case:

#### When to Use Semantic Schedules

✅ **Use semantic schedules when:**

- You need common business schedules (daily, weekly, monthly)
- Code readability is important
- Working with AI-assisted development
- Onboarding new developers
- Building business workflows

```typescript
// Clear and readable
every($.Daily, handler)
every($.BusinessDay, handler)
every($.Monthly, handler)
```

#### When to Use Cron Patterns

✅ **Use cron patterns when:**

- You need precise timing (e.g., 2:37am)
- Working with unusual schedules
- Migrating from existing cron jobs
- Need fine-grained control

```typescript
// Precise control
every('37 2 * * *', handler) // 2:37am daily
every('0 9,14,17 * * 1-5', handler) // 9am, 2pm, 5pm weekdays
```

### Semantic to Cron Mapping

| Semantic           | Equivalent Cron | Description      |
| ------------------ | --------------- | ---------------- |
| `$.Minutely`       | `* * * * *`     | Every minute     |
| `$.Every5Minutes`  | `*/5 * * * *`   | Every 5 minutes  |
| `$.Every15Minutes` | `*/15 * * * *`  | Every 15 minutes |
| `$.Hourly`         | `0 * * * *`     | Every hour       |
| `$.Daily`          | `0 0 * * *`     | Midnight daily   |
| `$.BusinessDay`    | `0 9 * * 1-5`   | Weekdays 9am     |
| `$.Weekly`         | `0 0 * * 1`     | Mondays midnight |
| `$.Monthly`        | `0 0 1 * *`     | 1st of month     |

## Advanced Scheduling

### Conditional Execution

Execute tasks based on conditions:

```typescript
import { $, every } from 'sdk.do'

every($.Daily, async () => {
  const today = new Date().getDay()

  // Only run on business days
  if (today >= 1 && today <= 5) {
    await businessDayTask()
  }

  // Only run on first day of month
  if (new Date().getDate() === 1) {
    await monthlyTask()
  }
})
```

### Dynamic Schedules

Create schedules based on data:

```typescript
import { $, every, db } from 'sdk.do'

// Load schedule configuration from database
const config = await db.get($.Configuration, 'report-schedule')

every(config.cronPattern, async () => {
  await generateReport()
})
```

### Multiple Handlers

Register multiple handlers for the same schedule:

```typescript
import { $, every } from 'sdk.do'

// Both handlers execute daily
every($.Daily, async () => {
  await generateReport()
})

every($.Daily, async () => {
  await cleanupOldData()
})
```

### Chained Schedules

Chain scheduled tasks together:

```typescript
import { $, every, send, on } from 'sdk.do'

// Primary schedule
every($.Daily, async () => {
  await send('DailyProcess.started')
})

// Triggered by event
on('DailyProcess.started', async () => {
  await step1()
  await send('DailyProcess.step1Complete')
})

on('DailyProcess.step1Complete', async () => {
  await step2()
  await send('DailyProcess.completed')
})
```

## Best Practices

### 1. Use Semantic Schedules for Standard Cases

```typescript
// Good: Clear intent
every($.Daily, generateReport)
every($.BusinessDay, sendReport)

// Less clear: Requires cron knowledge
every('0 0 * * *', generateReport)
every('0 9 * * 1-5', sendReport)
```

### 2. Document Unusual Cron Patterns

```typescript
// Good: Explained
// Run at 2:37am for off-peak processing
every('37 2 * * *', async () => {
  await heavyProcessing()
})

// Bad: No explanation
every('37 2 * * *', heavyProcessing)
```

### 3. Use Timezones Explicitly

```typescript
// Good: Explicit timezone
every($.BusinessDay, handler, {
  timezone: 'America/New_York',
})

// Risky: Assumes UTC
every($.BusinessDay, handler)
```

### 4. Consider Execution Time

```typescript
// Good: Off-peak hours
every('0 2 * * *', async () => {
  await heavyBackup()
})

// Bad: Peak hours
every('0 13 * * *', async () => {
  await heavyBackup()
})
```

### 5. Avoid Too Frequent Schedules

```typescript
// Questionable: Every minute might be too frequent
every($.Minutely, expensiveTask)

// Better: Batch and run less frequently
every($.Every5Minutes, async () => {
  await processBatch()
})
```

## Testing Schedules

### Test Schedule Logic

```typescript
import { $, every } from 'sdk.do'

// Export handler for testing
export const dailyReportHandler = async () => {
  const report = await generateReport()
  return report
}

// Use in schedule
every($.Daily, dailyReportHandler)

// Test independently
test('daily report generates correctly', async () => {
  const report = await dailyReportHandler()
  expect(report).toBeDefined()
})
```

### Validate Cron Patterns

```typescript
import { validateCronPattern } from 'sdk.do'

// Validate pattern before using
const pattern = '0 0 * * *'
if (validateCronPattern(pattern)) {
  every(pattern, handler)
} else {
  console.error('Invalid cron pattern')
}
```
